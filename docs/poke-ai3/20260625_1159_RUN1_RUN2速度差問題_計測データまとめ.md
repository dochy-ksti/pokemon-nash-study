# RUN1/RUN2 速度差問題 — 計測データまとめ

作成: 2026-06-25 11:59 JST

本書は「train-loop を同一スクリプト内で連続実行すると 1 本目(RUN1)は速く、
2 本目以降(RUN2…)は約 2 倍遅くなる」現象について、再現方法と計測した
客観データのみを記録する。考察・原因仮説は含めない。

---

## 1. 現象

- 同一スクリプト内で `uv run train-loop` を同一引数・別プロセスで連続実行する。
- 各プロセスは実行前に同一の checkpoint(`shared_init.pt`)を `cp` で上書きしてから開始する。
- RUN1 は速く、RUN2 以降は約 2 倍遅い。RUN3/RUN4 も RUN2 と同程度に遅い。
- プロセスは毎回新規(共有メモリ・残留スレッドなし)。

---

## 2. 再現方法

### 2.1 共通の train-loop 引数

```
uv run train-loop --num-games 32 --sim-concurrency 16 --sims 64 \
  --search-turn-min 4 --search-turn-max 7 --depth-skew 2.0 \
  --no-random --no-crit --stage 3b --max-epochs <EPOCHS> \
  --checkpoint-path <OUT>
```

- 各 run の前に `cp data/poke-ai3/tournament/shared_init.pt <OUT>` を実行。
- `battle_seed` は未指定。`train_loop.py:142` で `secrets.randbits(64)`(プロセスごと乱数)。

### 2.2 再現スクリプト(本調査で使用、`poke-ai3-python/scripts/`)

- `diag_fix.sh` — 4 連続 run の s/ep を出力。
- `diag_ipc.sh` — `perf stat` で 2 連続 run の命令数/サイクル/キャッシュ等を比較。
- `diag_lk.sh` — `POKE_AI3_ROOT_DIAG=1` で lookahead カウンタ(decisions/rollouts/plies/forced)を 2 run 比較。
- `diag_perf_both.sh` — RUN1/RUN2 を `perf record -F 499 -g --call-graph dwarf` で記録し flat self を出力。
- `diag_perf_fp.sh` — `RUSTFLAGS=-C force-frame-pointers=yes` で再ビルドし `--call-graph fp` で RUN1/RUN2 を記録。

### 3.6 perf record — シンボル化に関する計測上の事実

- dwarf 記録(diag_perf_both.sh)では、RUN1 の `_native` 最上位が
  `__FRAME_END__+0x3632a`(21.32%)。`addr2line` で解決不能(`??`)。
  RUN2 では同一 .so(build-id `ad7b2537…` で両 data 一致)で関数名へ解決される。
- fp 記録(diag_perf_fp.sh)では、RUN2 はワーカースタックが
  `tokio…poll → rollout` まで解決(86.98%)。RUN1 は `__FRAME_END__+0x3d28a`(36.80%)や
  生アドレス `0x73791b…` が並び、関数名へ解決されない。
- 両 data が参照する `_native` の build-id は一致:
  `ad7b2537354ef7dc5b6a290fed5602e4f1eb73d5`。

## 4. 切り分け済み(本現象との関連が否定された/確認された事実)

過去セッションで以下を計測・確認済み(本書では結論のみ列挙):

- CPU クロック: 遅い RUN の方がクロックは高い(throttling ではない)。
- 熱/電力・カーネル/THP・iowait: 速度差と相関なし。
- `time.sleep(0)` busy-poll の有無: 速度差に無関係。
- HashMap のランダム seed 衝突: `global_ids` を SipHash → FxHash(seed 固定)に変更しても
  速度差は解消せず(現象は乱数 seed 依存ではなく決定的に RUN1 速・RUN2 遅)。
- lookahead ループ回数(3.3): 両 run ほぼ一致。
