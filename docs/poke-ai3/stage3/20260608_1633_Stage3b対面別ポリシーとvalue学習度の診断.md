# Stage3b 対面別ポリシー・value学習度の診断 (ハンドオフ)

## このセッションでやったこと

`stage3b_long.pt`(300ep)の交代率が ~75% で頭打ちした件を、対面ごとに精査した。
新しい診断ツール `inspect-matchup` を作り、チェックポイントから自己対戦を回して
(学習・上書きせず)対面×HP別に「モデル policy / 教師 target_pi / selection_pi /
実際の着手 / lookahead 手別勝率 / 盤面勝率 value(model/target/realized)」を集計した。

### 対面分類

(active の技 × 相手 active 種族) の4対面。種族ベース(Cl_v_Go)では active が
Shock Wave 持ちか Bulldoze 持ちか区別できないため、実際に撃てる技で分類する。

- `SW_v_Cl` = Shock Wave vs Cloyster (SE/有利)
- `SW_v_Go` = Shock Wave vs Goodra   (半減/不利)
- `BD_v_Cl` = Bulldoze   vs Cloyster (等倍)
- `BD_v_Go` = Bulldoze   vs Goodra   (SE/有利)

## わかったこと(結論)

1. **学習ギャップは無い**。policy も value も model ≈ target_pi/target_value。
   75% は lookahead 均衡をモデルが忠実に再現した結果で、学習不足ではない。

2. **交代判断を決めるのは「active が SE を通せるか」だけ**。半減/等倍の区別は無関係。
   - SE 対面 (SW_v_Cl, BD_v_Go): 交代 ~0.21–0.28(攻撃を選ぶ)
   - 非SE 対面 (SW_v_Go=半減, BD_v_Cl=等倍): 交代 ~0.74–0.76(SEで殴れる相方へ交代)
   ユーザー仮説の「半減 vs 等倍の非対称性が交代率を下げている」は否定。
   等倍 BD_v_Cl(0.758)も半減 SW_v_Go(0.739)とほぼ同じ交代率。

3. **モデルは lookahead の手別勝率差(diff=交代−攻撃)に忠実**。
   diff が大きいセル(中HP帯, +0.2〜0.3)は交代率 ~1.0、diff≈0/負のセル(開幕満タン)は
   交代率が低い。開幕満タンの低交代は誤りではなく、lookahead 自身が
   「そこでは交代に勝率優位なし」(SW_v_Go −0.07 / BD_v_Cl +0.03)と判定した結果。

4. **75% 頭打ちの正体は HP状況の混合**。非SE対面の交代率は一様75%ではなく、
   開幕満タン(my_hi/opp_hi, n大)は低く(0.33/0.45)、被弾後は0.8〜1.0。集計0.75はその平均。

5. **value はモデル↔教師ギャップ無し**だが **target 自体が +0.09 の楽観バイアス**
   (target_value = 手別勝率の最大値=最善手前提)。集計 model ~0.60 / realized ~0.51。

6. **序盤 value は対面でほぼ平坦(~0.6)**。ただし自己対戦の開幕は実勝率も ~0.5 なので、
   平坦は欠陥でなく自然。対面の有利不利は「序盤勝率差」ではなくポリシー(不利なら交代)と
   中盤以降の value 差に現れる。

## 割引報酬(時間割引)の議論

ユーザー提案: 遠いターンの敗北を近い敗北より高く、近い勝利を遠い勝利より高く報酬付けして、
無差別局面でも交代に勾配を出す。現状 1.0/0.0/0.5 + 割引なし([lookahead.rs:158-169])。
`nash_accumulation` は無差別(win_rate フラット)だと diff≈0 → training_pi=prior をそのまま
返すだけで勾配を足さない([nash.rs])。

[0,1] を保ったまま両狙いを満たす形: `discounted = 0.5 + gamma^t * (terminal - 0.5)`。
value head の出力域も nash 閾値も触らず、lookahead.rs の末端 return と leaf 評価の2箇所 +
LookaheadConfig に gamma 追加で済む。

**ただし現データ上、強さのために割引を入れる根拠は無い**。低い開幕交代率は勝率に忠実な正解で、
無理に上げると勝率優位のない手(SW_go では負の手)を選ばせ、通常状態を弱体化させる
(=水平線効果の懸念どおり)。ユーザー見解: 序盤の勝率差が薄いのは自然で、終盤→中盤→序盤へ
value が伝播すれば序盤も尖ってくるはず(時間がかかる)。これは propagation 仮説として妥当。

## 実行方法

```bash
cd poke-ai3-python
uv run inspect-matchup --checkpoint-path ../data/poke-ai3/stage3b_long.pt \
  --stage 3b --num-games 8 --num-batches 4 --sims 64 --sim-concurrency 8 --no-random --no-crit
```

学習せず(チェックポイント非上書き)、自己対戦の trajectory を集めて対面×HP別に集計・表示する。

## 次のセッションでやること

1. **value calibration を標本を増やして実測**。今回は ~32試合相当で各HPセルが薄く、
   セル単位の realized がノイジー。試合数を増やして対面×HP別の `model_value − realized`
   (過大/過小評価マップ)をきれいに見る:

   ```bash
   uv run inspect-matchup --checkpoint-path ../data/poke-ai3/stage3b_long.pt \
     --stage 3b --num-games 16 --num-batches 16 --sims 64 --sim-concurrency 16 --no-random --no-crit
   ```

2. **ルールベース最善手エージェントとの対戦**。ユーザーが最善手とみなす
   「不利対面→交代 / 有利対面→攻撃」の決定論エージェントを実装し、学習済み AI と対戦させて
   勝率を測る。学習方策が本当に最善手相当に到達しているか(交代タイミングの妥当性)を、
   自己対戦均衡とは別の固定ベースラインで検証する。
   - 実装場所候補: 評価用の固定方策を Rust 側 game_task の片側、または推論時に policy を
     ルールで上書きする経路。`win_rates` / 対面分類ロジック(`_se_move_for` 等)を流用できる。
   - 注意: 自己対戦均衡が ~0.75 交代なのは「相手も混合で応手する」前提。固定ルール相手なら
     最適応手は変わる(相手が必ず交代/攻撃するなら、こちらの最適も決定論寄りになりうる)。
     この対戦で「学習 AI が固定最善手にどれだけ勝てるか / 搾取されないか」を見る。

3. (任意) 割引報酬 `0.5 + gamma^t*(term-0.5)` を gamma≈0.98〜0.99 で試作し、序盤交代率と
   対固定ルール勝率が変わるか A/B。ただし通常状態の強さを削らないか厳重に確認する。
