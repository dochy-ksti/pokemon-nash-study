## 対戦所感 (人間がプレイして体感)

- **不利対面交代 / 有利対面攻撃だけのナイーブ戦法では全く勝てなかった。**
- タイプ相性の問題で **Bulldoze が強い**。Shock Wave は撃たず、釣り交換して Bulldoze を
  通すのが強い (後述の非対称性が原因)。
- 体力が削れてくると **AI は釣り交換読み行動を交ぜてくる**。そこで **釣り交換読み攻撃**を
  決めると人間でも勝てる。
- **これだけの少要素 (2種族・2技・型相性・交代・同速コイン) でもポケモン対戦は非常に難しく
  なる**ことが体感できた。記憶なし・相手控え盲目の AI がここまで手強いのは均衡混合戦略の力。

## Bulldoze 非対称性の原因 (確認済み, 次タスク3の根拠)

`poke-sho-rust/src/types.rs:114` の Electric 攻撃テーブル:
```rust
Electric => match defending {
    Flying | Water => Some(1),
    Grass | Electric | Dragon => Some(-1),   // ← Dragon が電気を半減
    Ground => None,
    _ => Some(0),
},
```
- Goodra-Hisui = **Steel/Dragon**。Shock Wave (電) vs Goodra = Steel 中立 × **Dragon 0.5** = **0.5倍 (半減)**。
- Bulldoze (地) vs Goodra = Steel 2倍 × Dragon 中立 = **2倍 (SE)**。Bulldoze vs Cloyster = 中立。
- Shock Wave vs Cloyster (Water/Ice) = **2倍 (SE)**。
- **非対称性**: Bulldoze は「狙った相手に SE / 別相手に中立」だが、Shock Wave は「狙った相手に
  SE / **別相手 (Goodra) に半減**」。Shock Wave 側だけが二重に不利 → Bulldoze が一方的に強い。

## 次にやりたいこと

### 1. 学習を速くするコマンドの工夫
学習が非常に遅い。sims を増やして賢くしつつ、sim_concurrency を増やして1試合を速く終わらせたい。
試したいコマンド (experiments/poke-ai3/20260608_1759 の発展形):
```bash
cd poke-ai3-python
uv run train-loop --num-games 32 --sim-concurrency 256 --sims 512 \
  --search-turn-min 6 --search-turn-max 12 --no-random --no-crit --stage 3b \
  --max-epochs 300 --chunk-threshold 8192 \
  --checkpoint-path ../data/poke-ai3/some_file_name.pt
```
- 注意・制約 (`poke-ai3-python/src/lib.rs`):
  - `sim_concurrency <= sims` (256<=512 OK)。`search_turn_min <= search_turn_max`。
  - `chunk_threshold` 既定 = `num_games * sim_concurrency / 2`。実験で
    `threshold ≈ num_games*W/2` がスループット最適、`num_games*W` だとバリアが深く最遅、と判明済み
    (32*256/2=4096 が既定。ここでは 8192 を明示指定 = やや深めのバッチ)。
  - 5分以上かかる見込みのコマンドは実行前に引数を説明すること (AGENTS.md)。300ep は長時間。
- 検討点: sim_concurrency を上げると1試合あたりの rollout 並列度が上がり試合は速く終わるが、
  GPU バッチが大きくなりメモリ・レイテンシとのトレードオフ。最適 chunk_threshold は要実測
  (experiments に1件ずつ記録)。

### 2. 相手控えポケモンの HP トラッキング
現状 AI は **相手の場 (active) の種族 + 量子化 HP% しか観測しない**。相手の控えは種族も HP も
完全に隠匿、かつモデルは1ターンのスナップショットを forward するだけで履歴記憶もない
(`poke-env-rust/src/observation.rs:93` `observation_for`、相手は active のみ)。
→ **相手控えの HP (と存在/瀕死) を観測に入れて AI を賢くしたい。**
- 改修対象:
  - `poke-env-rust/src/observation.rs`: `StateForPlayer` に相手控えスロット (隠匿版 BenchSlot:
    種族 + 量子化 HP% + alive、技は隠す等) を追加。`observation_for` で相手 party を量子化して詰める。
  - `poke-ai3-python/python/poke_ai3_train/encoding.py`: `NUM_SLOTS` と入力幅を拡張
    (相手控えスロット分)。Rust と Python のスロット定義を必ず一致させる。
  - **観測次元が変わる → 既存 checkpoint は非互換。再学習が必要。**
  - 「履歴トラッキング」を真にやるなら、量子化 HP は Showdown 流に「一度見えた相手控えの最終 HP%を
    保持」する設計が要る。Local 真状態から作るなら素直に現在 HP を入れられるが、Showdown 互換の
    隠匿条件 (見ていない控えは不可視) とどう揃えるかは要設計判断。

### 3. Shock Wave / Bulldoze を完全対称にした対戦
型相性の非対称 (上記) を消し、完全対称な状態でも対戦してみたい。
- **暫定改修**: `poke-sho-rust/src/types.rs:114` の Electric テーブルから `Dragon` を外す:
  ```rust
  Grass | Electric => Some(-1),   // Dragon を削除 → 電気 vs Goodra(Dragon) が中立に
  ```
  これで Shock Wave vs Goodra = Steel 中立 × Dragon 中立 = **1.0倍 (中立)** となり、
  Bulldoze vs Cloyster = 中立 と対称になる。両技とも「狙った相手に SE / 別相手に中立」で完全対称。
- 注意: これは型相性表の改変なので、`poke-sho-rust` の他テスト (parity 検証等) への影響を要確認。
  一時実験用なのでフラグ化 or ブランチ分けが無難。対称化したら再学習して GUI で対戦比較。

- タスク2・3はいずれも **観測次元 or ルール変更 → 既存 checkpoint 非互換 → 再学習必須**。
