# タイプ相性導入と Stage3a (4技1v1) の実装

JST: 2026-06-05 14:54

## 目的

`poke-sho-rust` にタイプ相性を導入し、AI が「相手の弱点を突く技」を選べるかを検証する
シナリオ (Stage3a) を追加した。最終目標は cloyster/goodra のサイクル戦・交代読みだが、
3段階に分けて進める。本セッションは **第1段階 (Stage3a)** を完了。

### 3段階の計画
1. **Stage3a (本セッション完了)**: 4技 (Crunch/Dark Pulse/Shock Wave/Bulldoze) を持つ
   Cloyster と Goodra-Hisui の 1v1。両陣営ランダムに種を選び、相手の弱点を突けるか検証。
2. **Stage3b (未着手)**: 各ポケモン1技のみの2チームを用意し、交代を学ぶか検証。
3. **Stage3c (未着手)**: それに Crunch/Dark Pulse を加え、交代読みを検証。

Stage3b/3c は **交代機構**が前提で、本セッションでは未実装 (下記「残課題」)。

## 設計 (grill-me で確定した合意)

### ポケモンとタイプ
- **Cloyster** (みず/こおり, 既存種族値): Def 特化・SpD 低。
- **Goodra-Hisui** (はがね/ドラゴン, 種族値 80/100/100/110/150/60): SpD 特化・Def 低。
  原種 Goodra (ドラゴン単) とは別の新種族として追加。既存 Stage2a/2b は原種のまま不変。

### 技 (すべて威力60・命中100・追加効果なし=決定論維持)
- **Shock Wave** (でんき・特殊): Cloyster(みず) に 2倍 + 低SpD で刺さる。対 Cloyster キラー。
- **Bulldoze** (じめん・物理): Goodra-Hisui(はがね) に 2倍 + 低Def で刺さる。対 Goodra キラー。
- でんき/じめんは両種のタイプと不一致 → **両 SE 技とも STAB が常に乗らず完全対称**。
- Crunch (あく物理80) / Dark Pulse (あく特殊80) は両者に等倍。

### ステータス (Stage3a, level50, 222/94 鏡像)
| | HP | Atk | Def | SpA | SpD | Spe |
|--|--|--|--|--|--|--|
| Cloyster | 157 | 117 | 222 | 117 | 94 | 90 |
| Goodra-Hisui | 157 | 117 | 94 | 117 | 222 | 90 |

- 低い方が 94 なのはヒスイ Goodra の Def 種族値 100 の下限制約 (原種の 81 は不可能)。
- シミュレータは EV 合計 510 を強制しないため、現行 Stage2a 同様 510 超の EV 配分で実現。
- 性格: Cloyster=Relaxed, Goodra-Hisui=Gentle(+SpD/-Def)。

### 評価方針 (今後)
- Stage3a: 相手 Cloyster→Shock Wave、相手 Goodra→Bulldoze の選択率 >90% を定量判定。
  同種対面では Crunch/Dark Pulse(威力80) を選ぶか対照確認。
- Stage3b: 不利先発での交代率。Stage3c: 交代読みはトラジェクトリ定性レビュー主体。

## 速度タイ RNG の根治 (LCG 全廃 → ChaCha8)

Stage3a 学習中に、弱点突き (correct_action_rate) が 1.0 に収束した直後から win_rate が
0 付近へ崩落する現象を観測。原因は速度タイ (両者 Spe=90 同速) の RNG 退化:
- 旧実装は LCG の **最下位ビット**で先手を決定。LCG bit0 は周期 2 で毎ターン反転。
- さらに seed が 4 連続整数の XOR で初期パリティが常に 0 → **全ゲームで先手順が
  `P2,P1,…` に固定**され、決定論収束後に P1 がほぼ全敗していた (戦略劣化ではない)。

対応 (コミット済み):
- `battle_chacha.rs`: `rand_chacha::ChaCha8Rng` ベースの `BattleChaCha` を新設。ダメージ
  乱数・急所・速度タイ (`first_player`)・rollout 深さ (`below`)・方策サンプリング (`unit`)
  を全ビット良質な 1 ストリームから供給。seed からの完全再現性は維持。
- `local_showdown.rs` / `lookahead.rs`: 旧 `LocalRng`/`LcgRng`・自前 `tiebreak` LCG を撤去し
  `BattleChaCha::first_player()` に統一。`rollout_rng.rs` は削除。
- 検証: 修正後は win_rate が収束後も ≈0.5 に正常化 (旧 LCG では 0 崩落)。詳細は
  `experiments/poke-ai3/20260605_1551_ChaCha8化でwin_rate崩落を根治.md`。
- 副産物: 公平な速度タイ基盤ができたので、後段の **速度学習マイルストーン**(実速度差)
  でもそのまま使える。

## 検証
- `cargo test --workspace` 全通過。新規テスト: タイプ相性のシナリオ関係、222/94 鏡像

## 残課題 (次の増分: 交代機構 = Stage3b/3c)

交代は state/observation/action 空間の再設計を伴うため別 PR とする。要改修:
1. **poke-sho-rust**: `BattleState` を2体パーティ+場のポケモン index 化。`Choice` を
   `Move | Switch` に。`legal_choices` に交代手を追加。`apply_turn` で交代解決 (交代は
   行動前、瀕死時の強制交代)。
2. **poke-env-rust**: 観測に控えの種族/HP・場のポケモンを追加。`local_showdown` の
   request/force-switch 対応。
3. **poke-ai3 / python**: 行動空間を「4技 + 交代スロット」に拡張。観測エンコーディングに
   控えスロット追加。
4. team データ: `stage3b_team1/2.txt` (各1技), `stage3c_team1/2.txt` (+Crunch/Dark Pulse)。
   両陣営ランダムにチーム割当・ランダム先発。
