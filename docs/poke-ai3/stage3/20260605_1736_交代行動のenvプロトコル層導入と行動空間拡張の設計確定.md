## 残作業: ステップ4(行動空間拡張の縦切り、1 コミットで緑着地)

行動表現を `MoveId`/`NUM_MOVES` → `Choice`/`ACTION_DIM = NUM_MOVES + (MAX_PARTY-1)` に拡張する。
Python の policy_head を壊すため、**下記を一括で変えて初めて緑に戻る**。中途コミット不可。

### Rust: poke-env-rust

1. `observation.rs`:
   - `pub const ACTION_DIM: usize = NUM_MOVES + (MAX_PARTY - 1);` を追加(MAX_PARTY は poke-sho-rust から)。
   - `StateForPlayer` 拡張: `my_active_moves: [MoveId; NUM_MOVES]`(空き sentinel)、
     `my_bench: [BenchSlot; MAX_PARTY-1]`(`{species:u8, hp_frac:f32, alive:bool, moves:[MoveId;NUM_MOVES]}`)。
     `legal_action_mask` を `Vec<bool>`(長さ `ACTION_DIM`)に。交代枠は相対控え index(active を除く控え順)。
   - `observation_for`: 控えスロット詰め(active を除いた生存/全控えを相対順に)、交代合法手を
     相対 index で mask へ。相手側は据え置き(隠匿)。
   - 相対控え index ↔ 絶対パーティ index 変換ヘルパを本ファイルに置き、lookahead/game_task が共用。
2. `lookahead.rs`: `MoveId` 前提を `Choice` / `ACTION_DIM` に一般化。
   - `legal_moves` → `legal_choices`(技+交代枠、相対 index)。
   - `sample_move`/`ucb_select`/`sum_win`/`counts`/`win_rates`/`avg` を `ACTION_DIM` 次元・行動 index 基準へ。
   - rollout 内で `Choice::Switch` をサンプルし `apply_turn` へ。**rollout 内の強制交代解決**:
     `apply_turn` 後 `any_forced_switch()` なら policy から forced 側の交代手をサンプルし
     `apply_forced_switches` で解決してから次 ply(local_showdown と同じ部品)。
   - `run_lookahead` 冒頭: root の `legal_choices` が 1 手なら短絡即決(rollout を打たない)。
   - root 候補生成を `legal_choices`(通常/強制を区別せず)に。
3. `nash.rs`: `NUM_MOVES` 固定を `ACTION_DIM` 次元へ一般化(配列長・legal の扱い)。
4. `oracle.rs`: `OracleOut.policy` を `[f32; ACTION_DIM]` へ。

### Rust: poke-ai3

5. `async_executor.rs`: `MAX_MOVE_LEN`(= NUM_MOVES)を `ACTION_DIM` に置換 or 新定数 `ACTION_DIM` を導入し
   `InferencedDataItem.policy`/`PlayerObservation` を合わせる。
6. `root_task.rs`: `TrajectoryItem.target_pi` を `[f32; ACTION_DIM]` へ。
7. `inference_client.rs`: fallback uniform policy・配列長を `ACTION_DIM` へ。
8. `game_task.rs`:
   - 通常ターンで lookahead 結果から交代込みの行動をサンプルし、**相対控え index → `SendInfo::Switch(絶対+1)`** へ変換して送信(技は従来通り `SendInfo::Move(showdown_slot)`)。
   - `Request::ForceSwitch` を**学習決定ノード化**(observe→lookahead→交代手着手、合法手1なら短絡)。
   - Showdown 経路(`battle_state()` 無し)の `StateForPlayer` 手組み立てを新スキーマ(控え・active技)へ。
     ※相手隠匿なので控えは自軍 event 履歴から。Stage3b 検証は local 経路中心で可。

### Python: poke-ai3-python

9. モデル `policy_head` 出力次元を `ACTION_DIM` に。観測エンコーダに控えスロット
   (種族・HP・alive・技マスク)と active 技マスクを追加。
10. 行動デコード/`legal_action_mask` 適用を `ACTION_DIM` に。`supervised.py` の `target_pi` 形状追従。
    既存 `draws` 集計はそのまま。
