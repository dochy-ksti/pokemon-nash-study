# stage 3c 設計合意 — 非対称性を除去した対称対面

作成: 2026-06-15 00:11 JST

## 背景・目的

stage 3b は両側 {Cloyster, Goodra-Hisui} の2体・各1技の交代学習シナリオ。
ここに非対称性があった:

- **Shock Wave (Electric)** は Goodra-Hisui (Steel/Dragon) に対し **0.5倍（半減）**
- **Bulldoze (Ground)** は Cloyster (Water/Ice) に対し **1倍（等倍）**

この「片方は半減・片方は等倍」という非対称が選択を難しくしていた。

stage 3c では、この非対称性のみを除去した**完全対称な対面**を作り、3b → 3c で
**ナッシュ均衡（混合戦略）がどう変化するかを観察する**。目的は「交代学習を強くする」
ことではない。3b からの差分は次の2点のみに限定する:

1. Goodra-Hisui → 通常 Goodra（純 Dragon）
2. 技タイプの対称化（Electric/Ground → Fighting/Fairy）

確定数 (3HKO)・耐久・素早さ・威力60 は **3b 据え置き**とし、純粋比較を可能にする。

### 観察仮説

- 3b は Bulldoze（Cloyster に等倍）が Shock Wave（Goodra-Hisui に半減）より優位で、
  「Shock Wave を撃たずに釣り交換する」条件付きの複雑な戦略に価値があった。
- 3c で完全対称化すると、この**タイプ非対称由来の複雑さは消える**。
- ただし威力60（3HKO）据え置きでダメージは蓄積するため、**HP 状況依存の混合戦略は
  残る**と予想する。「対称化で消えるのはタイプ非対称の偏りであって、3HKO 由来の HP
  依存の読み合いは残る」ことを確認するのが本実験の主眼。

### 比較対象・チェックポイント

- 比較は **weak（`nash_accumulation_weak`）で学習した checkpoint 同士**で行う
  （通常 nash の「2329 ピーク」は使わない）。
- NUM_MOVES 4→6 で旧 3b weak ckpt（dim4）は新バイナリにロードできないため、
  **3b を新バイナリ（dim6）で再学習**し、3c と完全同形・同条件で比較する（方針C）。
- 3b/3c は**同 seed・同プロトコル**でペア学習する
  （hidden128 / no-random・no-crit / sims64 / search 6-12 / num-games 32）。
  まず **seed1** で1本ずつ、鏡像の一致や結論が割れたら **seed2 をペアで追加**。
- 再学習 3b には既存の対面別 diagnostics を当て、**旧 3b weak と定性傾向が一致するか
  サニティ照合**する（未使用2枠追加による交絡がないことの裏付け。旧 ckpt は参照値）。

## 対戦構造（3b と同型・ミラー割り当て）

- 両側 {Cloyster, 通常Goodra} の2体・各1技。チーム (Team1/Team2) と先発は独立ランダム。
- **team1**: Cloyster @ FightSpe60 / Goodra @ FairyPhy60
- **team2**: Cloyster @ FairyPhy60 / Goodra @ FightSpe60

## 新設技

| 技 | タイプ | 分類 | 威力 | 命中 | id |
|---|---|---|---|---|---|
| FightSpe60 | Fighting | Special | 60 | 100 | fightspe60 |
| FairyPhy60 | Fairy | Physical | 60 | 100 | fairyphy60 |

いずれも合成名（命名規則を統一）。Aura Sphere / Dragon Claw は当初案だったが、
前者は未実装、後者は Goodra (Dragon) が持つと STAB が乗り非対称が復活するため、
両方を合成技に置き換えた。

## 能力値（実数値, Lv50）

| | HP | Atk | Def | SpA | SpD | Spe |
|---|---|---|---|---|---|---|
| Cloyster | 157 | 117 | **222** | 117 | **94** | 90 |
| Goodra   | 157 | 117 | **94**  | 117 | **222** | 90 |

HP/Atk/SpA/Spe は同値、Def と SpD だけ反転する対称設計。通常 Goodra の spread
(stage3a_goodra_plain) は Goodra-Hisui と実数値が完全一致するよう作られている
（種族ベースが違うため EV/IV/性格で調整済み。素早さも90で一致）。

## 対称性の根拠

| | FightSpe60 (Fighting/特殊/60) | FairyPhy60 (Fairy/物理/60) |
|---|---|---|
| 弱点対象 | Cloyster (Ice) ×2 | Goodra (Dragon) ×2 |
| 等倍受け | Goodra (SpD 222) で受け | Cloyster (Def 222) で受け |
| STAB | 無（Cloyster/Goodra どちらが持っても不一致）| 無（同左）|
| 弱点ダメージ | 攻117 vs 弱点側耐久94, STAB無 | 同左 |

- **物理/特殊スプリット**と **Def↔SpD 反転 (94↔222)** が噛み合い、2倍ヒットも
  等倍受けも左右で完全一致する。
  - FightSpe60(特殊) は弱点 Cloyster の低 SpD 94 を叩く
  - FairyPhy60(物理) は弱点 Goodra の低 Def 94 を叩く
- STAB はシミュレータに実装済み (damage.rs:86) だが、両技とも保持個体とタイプ不一致
  のため乗らない。
- Fairy は Dragon に2倍・Water/Ice に等倍、Fighting は Ice に2倍・Water/Dragon に
  等倍（types.rs で確認済み）。

## ダメージ確定数（Showdown 準拠の式で算出）

威力60・攻撃117・弱点側耐久94・STAB無・急所無・2倍:

- base = ⌊⌊22×60×117 / 94⌋ / 50⌋ + 2 = 34、roll適用後 ×2 = **56〜68 ダメージ**
- HP157 に対し最大ロールでも 68×2 = 136 < 157 → **3HKO**

等倍受け（耐久222）: 約 17〜20 ダメージ（HP の約 11〜13%）と非常に軽い。

（参考: 威力80 にすると概ね 2HKO になりテンポが変わるため、3b 踏襲で威力60 を採用。）

## 実装項目

1. **team データ** `team/poke-ai3/scenario/`
   - stage3c_team1.txt / stage3c_team2.txt 新設。
   - 記載順は **Cloyster → Goodra**（宣言順スロット配置のため）。
   - Cloyster は stage3a_cloyster の spread、Goodra は stage3a_goodra_plain の spread を流用し、技を単一化。
   - 特性 Illuminate、持ち物 Covert Cloak、Tera は表記維持（新技に追加効果が無く無害）。
   - **実施済み**: stage3 全 team ファイルの特性を Illuminate に統一（Shell Armor 等は
     特性未実装下のバグだったため）。

2. **moves.rs**
   - FightSpe60 / FairyPhy60 を新設、`move_by_name` に追加。

3. **scenario.rs**
   - `MoveId` に2 variant 追加 → `NUM_MOVES = 6`（既存4技据え置き、index 4,5 に追加）。
     `data` / `index` / `from_index` / `ALL` / `from_showdown_id` を更新。
   - `Stage::Stage3c` 追加（short_name "3c"、is_party=true、from_short_name 対応）。
   - `TeamId::team_text` を **stage 引数化**し、Stage3c では stage3c team を返す。
   - `build_team_party` を `slots[sid.index()]` から **宣言順配置**へ変更
     （通常 Goodra は SpeciesId::Goodra=2 で MAX_PARTY=2 を超え overflow するため）。
     3b は team ファイル先頭が Cloyster のため挙動不変。stage 引数を伝搬。

4. **テスト**: 3c の技割当・スロット配置・対称性のテストを追加。

5. **Python 評価系の stage パラメータ化**
   （diagnostics.py / eval_vs_rule.py / eval_ckpt_*.py）
   - `_se_move_for` 等を stage 分岐（3a/3b: ShockWave→Cloyster, Bulldoze→GoodraHisui。
     3c: FightSpe60→Cloyster, FairyPhy60→Goodra）。
   - `--stage` choices に "3c" 追加。技ラベル・rule-agent 方策・assert を 3c 対応。

6. **観測次元**: NUM_MOVES 4→6 で観測/行動次元が拡大するため、stage3c は**新規学習**
   （3a/3b の旧 checkpoint とはネットワーク形状が非互換）。比較のため **3b も dim6 で
   再学習**する（方針C）。旧 3b weak ckpt はロード不能なので参照値扱い。

7. **人間対戦基盤の stage3c 対応**（おまけ・diagnostics で結論を出した後の後段作業）
   - `human_game.rs` の `Stage::Stage3b` 決め打ち（[human_game.rs:77](../../poke-ai3-python/src/human_game.rs#L77)）を
     **stage 引数化**し、`play_gui.py` 側でも 3c を選べるようにする。
   - 強制交代の一択ロジック（控え1体前提）は 3c でもそのまま流用可。
   - **同時手番**（人間の選択確定前に AI の手が見えない）を維持すること。
   - lead/active は全箇所で**宣言順インデックス**前提に統一（実装項目3の宣言順配置と整合）。

8. **diagnostics の HP 層別拡張（本実験の主役）**
   - 交換/技選択の確率を **HP 5段（満タン / 75%以上 / 50%以上 / 25%以上 / 25%未満）×
     自分 active 5 × 相手 active 5 のフル層別**で出す。既存の対面別キーに HP バケットを追加。
   - **unfold**（鏡像セルを分けたまま）で集計し、鏡像セルの一致＝対称均衡の再現を所見にする
     （3b は非対称なので unfold のみ）。
   - 学習ループ内の毎 epoch 診断は軽量な既存指標のまま。フル 5×5 は**収束 ckpt 固定の
     専用大サンプル観察スクリプト**で取得し、3b/3c 両 ckpt に同じものを当てて並べる。
   - 各セルに最小 n ガードを設け、枯れるセル（瀕死×瀕死 等、self-play で自然発生しにくい局面）は
     **正直に欠測扱い**とし確率を主張しない。

## 評価方針

本実験は「強さ比較」ではなく「均衡構造の観察」なので、評価軸を以下に置く:

- **主役 = diagnostics の HP 5×5 層別（実装項目8）**。これで結論（HP 依存の混合が
  3c でも残るか）を出す。3b（再学習・dim6）と 3c を同 seed・同形で並べる。
- **おまけ = 人間 vs AI 対戦（実装項目7）**。均衡の質感の変化を定性的に体感・記述する
  だけで、結論の根拠にはしない。
- weakness_rate は混合戦略ナッシュのため「正解率」ではなく**参照値**。rule-based 勝率や
  ckpt 直接対戦の絶対値も、対称化で天井が動くため**強さの主指標には使わない**。

| | 手段 | 役割 |
|---|---|---|
| 定量（結論） | diagnostics HP 5×5（unfold）| HP 依存構造の有無を確定。鏡像一致で対称均衡の再現を検証 |
| 定性（補強） | 人間 vs AI 対戦 | 読み合いの質感の変化を体感・記述（おまけ）|

## 設計判断の経緯（要約）

- 当初は Aura Sphere / Dragon Claw 案 → Dragon Claw は Goodra に STAB が乗り非対称が
  復活、Aura Sphere は未実装。→ 両方を STAB の乗らない合成技 (FightSpe60/FairyPhy60)
  に置換し、ミラー割り当てのまま完全対称を達成。
- 威力は 3b 踏襲で 60（3HKO 維持）。
- MoveId は拡張案A（NUM_MOVES=6、新規学習）を採用。
- パーティスロットは宣言順配置に変更（species index 依存を解消）。
- 実験のゴールを「交代学習の改善」から「**ナッシュ均衡の変化観察**」に再定義。主役は
  diagnostics（HP 5×5・unfold）、人間対戦はおまけ。
- 3b/3c 比較は dim を揃えるため **3b を dim6 で再学習**（方針C。案A=別バイナリ並置、
  案B=NUM_MOVES の stage 依存化 は不採用）。同 seed ペア学習、seed1 から開始。
- 手番順は素早さでなく公平コイン（ChaCha8）であり `--no-random` でも対称性が保たれる
  ことを確認済み（先手固定化バグは修正済み）。同時手番も維持。
