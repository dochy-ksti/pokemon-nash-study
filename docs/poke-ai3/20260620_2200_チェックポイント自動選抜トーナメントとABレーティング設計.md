# チェックポイント自動選抜トーナメント + A/B レーティング設計

作成: 2026-06-20 22:00 JST

## 背景・目的

これまでの A/B テスト（例: `depth_skew` の 4-8 skew2.0 vs 6-12 skew1.0）では、
「ピークの強さになったタイミング同士を head-to-head で比較」していたが、
**どの checkpoint がピークかを恣意的に選んでいた**。これをシステマティックに
決定できるようにする。

最終目的: 2つの学習手法 A / B のどちらが優れているかを、人手の選定なしに
レーティングで判定する。

---

## 1. 高速 head-to-head（新規 Rust モード）

現状の head-to-head（`eval_ckpt_vs_ckpt.py`）は executor 内 `run_lookahead` で
sims 分の value rollout を回しており重い。これを廃し、高速化する。

- **policy-only モードを新設**: 各手番で value net rollout を一切回さず、
  policy net 出力を既存 `sample_choice` に通して **確率で1手サンプル**し、
  終局まで進める。先読みは完全廃止する。
  - 確率サンプリングのみ（argmax にしない）。本 AI は Nash 混合戦略を学習して
    いるため、argmax で潰すと混合戦略が崩れ実力を測れない。
- **乱数は学習と揃えて off**（`--no-random --no-crit`）。policy の確率サンプリング
  だけで 1024 試合に十分な多様性が出る。本番 Showdown 用の頑健性は別フェーズ。
- `lib.rs` は現状 `sims == 0` を弾くため、policy-only 経路は別途追加が必要。

### 対戦・判定ルール

- **1024 試合／先攻 512・後攻 512** に分割（先手後手バイアス対策）。
- **勝率 50% 超で勝ち**。引き分け（ターン上限）は勝ち数比較から除外。
- 閾値はまず素の 50% で運用する。弱いはずの checkpoint が最後まで選ばれ続ける
  ようなら、マージン付き判定（例: 0.525 超で勝ち／互角帯は実力差なし）を検討する。

---

## 2. funnel（多段選抜）

funnel 各段のサイズは**可変パラメータ**にする。まず縮小版から開始し、
すぐ終わるようなら元案まで増やす。

| 段 | 縮小版（開始） | 元案 |
|----|----------------|------|
| ピーク収集数（0回戦突破） | 3 | 5 |
| 二次予選通過 目標数 | 2 | 3 |

### 訓練と評価の関係（重要）

- 訓練は **1本の連続ラン**。`--snapshot-every 5` で ep5/ep10/ep15… を最新重みから
  吐き続ける。
- **訓練前進と評価選抜は完全分離**。funnel の「敗者削除」は評価プールから外すだけで、
  訓練の重みは常に最新から前進し続ける（古い重みへロールバックして訓練し直したり
  はしない）。h2h は「どの checkpoint を評価用に残すか」を決めるだけ。

### 0回戦（隣接ピーク検出）

- 隣接 2 checkpoint を h2h。**敗者は常に削除**。
- **旧が勝ったら旧をプール昇格**（＝後続より強い＝ローカルピーク）。
- 新が勝ったら新がチャンピオンとして前進（昇格なし、＝まだ上り坂）。
- **ピークを1個拾うごとにリセット（方式A）**: 昇格した checkpoint は現役から外し、
  比較相手が居なくなるので次の checkpoint が新チャンピオンになる。
  → ピークが非重複区間から拾われ、同じ checkpoint の二重昇格が起きない。状態は
    `current` 1個だけで済む。

### 一次予選

- ピーク（0回戦突破）が **3個**たまる → 3者総当たり 1024 試合、**勝率1位**が
  一次予選通過、残りは削除。

### 二次予選

- 一次予選通過が **2個**たまったら、同じ隣接ピーク検出（方式A）を一次予選通過
  レベルで実施。旧が勝てば二次予選通過に格上げ、新が勝てば敗退削除。
- **二次予選通過を 2個（縮小版）**作る。

### A/B 両方で実施

- A / B それぞれで上記 funnel を完走 → 最終プール **2(A) + 2(B) = 4 checkpoint**。

---

## 3. 最終レーティング

- 4者**総当たり**（C(4,2)=6 ペア）、各 **1024 試合・先後 512/512**。
  手法をまたぐペアも同手法内ペアも全て戦う（グラフ連結のため）。
- **Bradley-Terry（ロジスティック MLE）**でレート推定。総当たりが完全なので
  順序非依存で安定。基準は平均を 0（または 1500）に固定して相対比較。
- **A 由来 2個の平均レート vs B 由来 2個の平均レート**。高い方が優れた手法。
- 各手法 2個平均のためサンプルは小さくノイズは残る。縮小版がすぐ終われば funnel
  サイズを元案（3／5）へ増やして安定化する。

## 補足・留保

- プラトーに達すると隣接ピーク判定の 50% 超は事実上コイントスになり、互角帯の
  checkpoint からノイズで拾うことになる。これは「弱いのが残ったら考える」で許容済み。
- 先読みを外すため「先読みすれば強い checkpoint」の差は測れない。policy 単体の
  強さで評価することを許容済み。
