# Stage3a (正解あり) での新旧エンコーディング A/B 比較

## 目的

前回の A/B 比較 (20260610_2133) は Stage3b で行ったが、Stage3b の攻撃/交代は混合戦略の
Nash 均衡であり一意の「正解」がない。Stage3a (1v1・4 技・タイプ相性) は「相手の弱点技を
選ぶ」という一意の正解があるため、`correct_action_rate` (greedy argmax の正解技率) を
真の正解率として新旧を比較する。

旧 = 197995d (1スカラ=1トークン + CLS 一括ヘッド)、
新 = d3ae347 (master、Embedding + フィールド単位トークン + pointer ヘッド)。

## コマンド

両側とも同一 (旧側は 197995d の worktree から実行)。決定論設定:

```bash
uv run train-loop --num-games 32 --sim-concurrency 16 --sims 64 \
  --search-turn-min 4 --search-turn-max 8 --no-random --no-crit --stage 3a \
  --max-epochs 5 --chunk-threshold 512 --battle-seed {1001,1002,1003}
```

追試として旧側 seed 1001 を `--max-epochs 15` で再実行。
ログ: `experiments/poke-ai3/logs_encoding_ab/{new3a,old3a,old3a15}_seed*.log`。

## 結果

correct_action_rate の epoch 推移:

| run | ep1 | ep2 | ep3 | ep4 | ep5 | 1.0 到達 |
|---|---|---|---|---|---|---|
| new seed1001 | 0.52 | 0.94 | 0.75 | 1.00 | 1.00 | ep4 |
| new seed1002 | 0.72 | 0.78 | 0.72 | 1.00 | 1.00 | ep4 |
| new seed1003 | 0.24 | 0.76 | 1.00 | 1.00 | 1.00 | ep3 |
| old seed1001 | 0.72 | 0.66 | 0.46 | 0.40 | 0.48 | 未到達 |
| old seed1002 | 0.53 | 0.47 | 0.52 | 0.54 | 0.57 | 未到達 |
| old seed1003 | 0.72 | 0.69 | 0.47 | 0.60 | 0.54 | 未到達 |
| old(15ep) seed1001 | 0.66 → 0.40〜0.79 を振動 | | | | ep15=0.45 | 未到達 |

- **新: 3/3 シードが ep3〜4 で 1.0 へ収束し、以後安定** (vs_cloyster/goodra とも 1.0)。
- **旧: 3/3 シードが 5 epoch で未収束。15 epoch へ延長しても 0.4〜0.8 を振動**し
  収束の気配がない。
- value_loss は両側同水準 (~0.04〜0.06)。差は policy の正解技学習に集中している。

## 結論

**新エンコーディングは Stage3a で旧より明確に速く・安定して学習する。**
合格 (同等以上) どころか系統的に優位。

解釈: 新方式は技トークンに「タイプ Embedding (種族と共有) + 威力 + 物理/特殊」を
直接持たせているため、タイプ相性と耐久の歪み (Cloyster=高Def/低SpD 等) を観測から
直結で学べる。旧方式は 1 スカラ=1 トークンの one-hot 群から相性構造を間接的に
復元する必要があり、5〜15 epoch では足りない。Stage3b (各個体 1 技) では技選択の
難しさが消えるため差が見えなかったが、技が 4 択になる Stage3a で差が顕在化した。

前回 (Stage3b) の「同等」と合わせ、スケール対応の設計判断
(Embedding + 構造化特徴 + pointer) は学習面でも正当化された。
