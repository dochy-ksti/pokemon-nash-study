# Phase 2 Stage 2a-det 400 epoch PPO 学習

- 開始 (JST): 2026-05-29 18:01
- 終了 (JST): 2026-05-29 18:04 (約 3 分、ビルド込み)
- ログ: [20260529_1801_phase2_stage2a_det_400ep.log](20260529_1801_phase2_stage2a_det_400ep.log)
- チェックポイント: `data/poke-ai3/phase2_stage2a_det/checkpoint.pt`

## 目的

Phase 2 Stage 2a (最大比率、Cloyster Def=222/SpD=81 ⇔ Goodra Def=81/SpD=222、
ratio ≈ 2.74x) 決定論モードで、AI が相手の防御特化方向を識別して物理 (Crunch) と
特殊 (Dark Pulse) を選び分けられることを再現性をもって確認する。前セッションで
ユーザーが 400 epoch で収束を観測したのを、`--num-games 8 --trajectories-threshold 8`
の設定で再走させる。

## コマンド

```bash
cd poke-ai3-python
uv run phase2-loop \
  --num-games 8 \
  --chunk-threshold 16 \
  --trajectories-threshold 8 \
  --max-epochs 400 \
  --no-random --no-crit \
  --backend local \
  --checkpoint-path ../data/poke-ai3/phase2_stage2a_det/checkpoint.pt
```

- `--no-random --no-crit`: 決定論モード (ダメージロール最大値固定、急所なし)
- `--num-games 8`: 4 種族マッチアップ × 並列度 2、観測の偏りを抑える
- `--trajectories-threshold 8`: 8 ゲーム揃ったら 1 PPO update。両種族の対戦が
  毎 epoch 含まれる頻度を上げる目的
- 相手 (P2) は Python 側で一様 `[0.5, 0.5]` を返すランダム相手 (Stage 2a の Q11)

## 構成

- モデル: Phase2PolicyValueModel (ModernBERT 隠れ 128 / 2 層 / max_pos=9)
- 観測スロット 8: my_hp, opp_hp, mask×2, my_species one-hot×2, opp_species one-hot×2
- PPO: lr=1e-4, γ=1.0, λ=0.95, clip=0.2, value_coef=0.5, entropy_coef=0.01,
  ppo_epochs=4, minibatch=64
- バトル: Lv50 gen9customgame、両者 Spe=90 同速、ターン上限なし、報酬±1/0
- 学習側固定 P1、相手 P2 は uniform random

## 結果

- 総 epoch: 400
- 初めて `correct_action_rate ≥ 0.95` を達成: epoch 22 (一時的、まだ揺れる)
- 連続 20 epoch `win_rate ≥ 0.95` 達成: epoch 103 (持続収束開始の目安)
- 末尾 100 epoch 平均:
  - **win_rate = 0.9717**
  - **correct_action_rate = 0.9407**
  - vs Cloyster Dark Pulse 選択率: **0.9378** (94 epoch で観測)
  - vs Goodra Dark Pulse 選択率: **0.0500** (93 epoch で観測 → Crunch 95%)
  - entropy = 0.2005
  - raw_logits_std = 1.5392

## 解釈

- AI は「相手 Cloyster → Dark Pulse、相手 Goodra → Crunch」を一貫して選べる
  ようになっており、Stage 2a-det の卒業基準 (連続 200 ep 勝率 95%) を満たす。
- Phase 1 で経験した soft-uniform 詰まり ([[project_raw_logits_soft_uniform_confirmed]])
  は再現せず、raw_logits_std は 1.5 前後まで健全に成長。種族 one-hot 観測の
  追加が効いていると考えられる。
- 末尾でも entropy が 0.20 残っている。エントロピー係数 0.01 と探索のための
  残留であり、stochastic な誤選択 (1/16 程度の頻度) を許容している。Stage 2b で
  必要なら entropy_coef を下げる検討余地あり。
- 収束タイミングは前セッション報告 (約 epoch 400 で収束) と整合的だが、本走では
  `trajectories-threshold=8` にして観測量を増やしたため、より早い epoch 100 付近
  から既に安定。
