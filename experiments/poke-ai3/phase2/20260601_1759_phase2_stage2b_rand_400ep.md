## 目的

直前に卒業した Stage 2b-det (ratio 1.57x、決定論モード) と同設定のまま、
`--random --crit` で 16 段ダメージロールと急所を有効化し (Stage 2b-rand)、
確率モードでも防御方向の識別学習が安定するかを確認する。マイルストーンの
「各決定論フェーズが動いた後に accuracy・急所・16 段ロールを足して学習が
安定するか試す」段階に相当。

## コマンド

```bash
cd poke-ai3-python
uv run phase2-loop \
  --stage 2b \
  --num-games 8 \
  --chunk-threshold 16 \
  --trajectories-threshold 8 \
  --max-epochs 400 \
  --random --crit \
  --backend local \
  --checkpoint-path ../data/poke-ai3/phase2_stage2b_rand/checkpoint.pt
```

- `--random --crit`: 16 段ダメージロール + 急所を有効化 (確率モード)
- 他は Stage 2b-det と同一 (`--num-games 8 --trajectories-threshold 8`)

## 構成

Stage 2b-det と同一 (モデル・PPO ハイパラ・観測スロット)。差分は `--random --crit` のみ。

## 結果 (ネガティブ)

- 総 epoch: 400
- `correct_action_rate ≥ 0.95` 達成: **なし**
- 連続 20 epoch `win_rate ≥ 0.95`: **なし**
- 末尾 100 epoch 平均:
  - win_rate = 0.708
  - correct_action_rate = 0.506 (≈ 当てずっぽう 0.5)
  - vs_cloyster_special_rate = 0.530
  - vs_goodra_special_rate = 0.515
  - entropy = 0.689 (≈ ln2 = 0.693、2 択一様)
  - raw_logits_std = 0.070
  - mean_return = 0.440

## 評価

- **soft-uniform 詰まりが再現**。`--random --crit` を入れた途端、policy は
  まったく学習せず entropy ≈ ln2 / raw_logits_std ≈ 0.07 の一様分布に固着した。
  correct_action_rate は 400 epoch を通して 0.5 付近から動かない。
- 決定論版 (Stage 2b-det: correct 0.995, raw_logits_std 2.94, entropy 0.021) と
  正反対の結果。狭い比率 (1.57x) では、ダメージのばらつき + 急所が報酬に
  ノイズを乗せ、物理/特殊の優劣シグナルが埋もれて勾配が育たないと考えられる。
- これは Phase 1 や Ver.6 で観測した soft-uniform 現象 (head が育たず logits std
  が小さいまま) と同型。Stage 2a (ratio 2.74x) の rand 版なら比率が広いぶん
  シグナルが残る可能性があり、切り分け候補。
- 次の方針候補:
  1. Stage 2a-rand (2.74x、確率) を同条件で走らせ、比率がノイズ耐性を決めるか確認。
  2. Stage 2b-det 卒業重みから --random --crit でファインチューン (転移で初期 logits
     を立たせてからノイズに晒す)。
  3. entropy_coef を下げる / 報酬のばらつきを抑える設計の見直し。
