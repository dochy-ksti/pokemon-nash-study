# Stage 3d 等倍打撃追加の厳密 Nash 比較

## 目的

Stage 3b の各個体に `Crunch`（かみくだく、物理80）と `Dark Pulse`
（あくのはどう、特殊80）を追加し、弱点技でない等倍打撃を選べると厳密 Nash 均衡が
どう変わるかを調べる。既存3bを保存し、新しい `Stage 3d` として比較した。

3d の各個体は次の3技を持つ。

- Team1 Cloyster: Crunch / Dark Pulse / Shock Wave
- Team1 Goodra-Hisui: Crunch / Dark Pulse / Bulldoze
- Team2 Cloyster: Crunch / Dark Pulse / Bulldoze
- Team2 Goodra-Hisui: Crunch / Dark Pulse / Shock Wave

種族・実数値・チーム間の弱点技割当・おんみつマントは3bと同一である。

## 均衡計算

3bと同じゲーム条件を使用した。

- HP buckets: 26
- 急所あり、16段階ダメージ乱数あり
- 毎ターン1%の幾何打ち切り (`discount=0.99`)
- 打ち切り時は生存数、次に残HP割合でタイブレーク
- cross-team only、全3,380,000有効状態

3dは最大4行動（3技＋交代）なので、従来の交代確率1値ではなく、各状態について
`[Crunch, Dark Pulse, coverage, Switch]` の完全方策を保存するようキャッシュ版ソルバを
一般化した。一般の3×3／4×4零和行列は support enumeration で解く。

```bash
cd poke-ai3-python
uv run python scripts/run_nash_geo_h26.py --stage 3d --hp-buckets 26
```

VIは支持集合境界の `f32` 微小振動を考慮して `vi_tol=5e-6`、固定方策評価と
best response評価は `eval_tol=1e-6` とした。保存精度は0.001である。本計算は399秒、
最大RSS 11,237,080KB。遷移キャッシュは372,782,808 entries。

結果:

- exploitability mean: `0.00000195`
- exploitability max: `0.00001514`
- 全有効状態の平均方策:
  - Crunch 25.67%
  - Dark Pulse 26.43%
  - coverage 25.51%
  - Switch 22.39%
- 2行動以上を2%超で混合する状態: 2,350,986 / 3,380,000

整合検証では、量子化後の方策和は全状態で999〜1001、控え瀕死時に交代確率が非0の状態は
0件だった。鏡像値 `V(s)+V(swap(s))` のu16誤差は最大1（実数0.001）、平均0.00794
（実数0.00000794）。

比較対象3b:

- exploitability mean `0.00001351`、max `0.00662506`
- 全有効状態の平均: coverage 52.32%、Switch 47.68%
- 同じ定義の混合状態: 1,108,783 / 3,380,000

## 結論

等倍打撃を追加すると「不利なら交代」は大きく崩れた。不利対面でも着手の52.2%を
Crunch/Dark Pulseに使い、交代は86.3%から45.4%へ低下した。有利対面でも弱点技一択には
ならず、相手の交代先へ通す等倍技を54.7%使用した。

その結果、全体の交代率は半分以下になり、平均対戦時間も約63%短縮した。一方で混合戦略状態は
約2.12倍に増えた。弱点でないが交代先にも一定ダメージを与える技はサイクルを止めやすいが、
戦略を単純化するのではなく、技カテゴリまで含む読み合いを増やす。

Web配信は今回の対象外。H=26テーブルと軌跡JSONは `data/poke-ai3/nash_geo/` に保存した。
