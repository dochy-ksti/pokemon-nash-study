# nash_learning_rate A/B — K1 r0.5 で 2.0 vs 1.5

策定/実行: 2026-07-07 (NLR20 07:06–08:02, NLR15 08:02–09:12, rate 18:xx JST)

## 結果サマリ (先頭掲載)

**nash_lr=1.5 が手法平均 +5.4 vs 2.0 の −5.4 (Δ≈10.8 Elo) で上回ったが、優位はほぼ
単一 finalist (NLR15#2 +18.7) 依存。各1本=同系譜3 finalist の run 運が支配的で有意判定は
不能。「1.5 が良いかも」の探索的示唆に留まる (既定 2.0 の変更は保留)。**

### Bradley-Terry (NLR20+NLR15 の全6 finalist 1プール総当たり, random+crit, n-per-side 512)

| finalist | nash_lr | 個体レート |
|---|---:|---:|
| NLR15#2 (ep570) | 1.5 | +18.7 |
| NLR15#0 (ep160) | 1.5 | −1.2 |
| NLR15#1 (ep365) | 1.5 | −1.4 |
| NLR20#2 (ep430) | 2.0 | −4.1 |
| NLR20#1 (ep290) | 2.0 | −5.5 |
| NLR20#0 (ep130) | 2.0 | −6.4 |

| 手法 | nash_lr | 手法平均 | 個体 |
|---|---:|---:|---|
| **NLR15** | 1.5 | **+5.4** | −1.2 / −1.4 / +18.7 |
| NLR20 | 2.0 | −5.4 | −6.4 / −5.5 / −4.1 |

### 結論
- 手法平均は NLR15 (1.5) が +10.8 Elo 上回るが、内訳は NLR15#2 の +18.7 一点依存で
  残り2 finalist は −1.2/−1.4 と NLR20 群にほぼ埋没。
- これは experiments 20260702_1312 (自己対戦比率掃引) が最終的に確認した罠と同型:
  **1 funnel run = 3 finalist は同系譜で相関し独立標本にならない**ため、run 単位ノイズ
  (battle seed + 学習確率性) が method-avg を支配する。
- したがって「nash_lr=1.5 が 2.0 より強い」は**有意ではない**。探索的に 1.5 側が良い可能性を
  示すのみ。**既定 nash_learning_rate=2.0 は現時点維持**。
- 厳密確定には各アームを 3 本ずつ (battle seed 違い) 複製し method-avg を独立標本化する必要がある。

## 目的・仮説

敵混合 K=1 / 自己対戦比率 r=0.5 (現行既定構成) を固定し、nash_weak 穏当化版の更新率
`nash_learning_rate` を軸に 2.0 (既定) vs 1.5 (やや穏当) を比較する。仮説: 更新率を下げると
Nash 追従が穏やかになり学習が安定して強くなるかもしれない。

## 実装 (本セッション)

- `ckpt_tournament.py` funnel に `--nash-learning-rate` (type=float, default=2.0) を追加し
  `_ensure_train_session` 経由で `TrainSession(nash_learning_rate=...)` へ伝達。従来 funnel は
  train-loop 既定 2.0 に固定されており nash_lr を振れなかった。
- 出力先 `TDIR` を旧 private 絶対パスのハードコードからリポジトリルート相対 (`data/poke-ai3`)
  へ修正 (`POKE_AI3_DATA_DIR` で上書き可)。本 A/B の初回実行はこの修正前に走ったため成果物は
  旧 private `data/poke-ai3/tournament/` に生成された (次回以降は public へ出る)。

## 手法 (1手法=1 funnel run→finalist 3)

| 手法 | tag | nash_lr | self-play-ratio | 実行 |
|---|---|---:|---:|---|
| NLR20 (既定) | NLR20 | 2.0 | 0.5 | 新規1 run |
| NLR15 (穏当) | NLR15 | 1.5 | 0.5 | 新規1 run |

- 両アームは `shared_init.pt` 共有・`--nash-learning-rate` 以外すべて同一。K1 r0.5 = 現行既定構成。

## パラメータ (NLR20 の例。NLR15 は --nash-learning-rate 1.5)

```bash
cd poke-ai3-python && make build && \
uv run python scripts/ckpt_tournament.py funnel --tag NLR20 \
  --start "$TDIR/shared_init.pt" --enemy-window 1 --self-play-ratio 0.5 \
  --nash-learning-rate 2.0 \
  --depth-skew 2.0 --search-turn-min 4 --search-turn-max 8 \
  --sims 64 --sim-concurrency 16 --train-num-games 64 --train-max-batch-size 512 \
  --train-trajectories-threshold 128 --train-minibatch-size 256 \
  --epochs-per-step 5 --train-block-epochs 50 --warmup-steps 10 \
  --peaks-per-rr 3 --finalists-target 3 --max-added-epochs 1000 \
  --n-per-side 512 --num-games 256 --stage 3b --random --crit
```

### 結論 (更新)
- 順位は **1.5 > 1.25 > 2.0**。`nash_lr=2.0` (既定) が3 finalist 全て下位 (−2.4/−5.1/−5.4) で
  揃い最下位、というのは**2回連続で 2.0 が負け**ており「既定 2.0 は下げた方が良さそう
  (1.25〜1.5 帯)」という方向性はそこそこコヒーレント。
- ただし **1.25 vs 1.5 の優劣は依然ノイズ**: NLR15 の +3.3 は NLR15#2(+11.7) 一点、
  NLR12 の +1.0 も NLR12#1(+6.0) 一点依存。各1本=同系譜3 finalist の相関という罠は未解消。
- **暫定示唆**: 既定 2.0 は最適でない可能性 (探索的)。ただし最適点 (1.25 か 1.5 か、
  あるいはさらに下) の確定には各アーム複数 seed 複製が必須。現時点で既定変更はしない。
- 成果物は3手法とも public TDIR に自己完結 (NLR20/NLR15 は private からコピー・パス書換済み)。
