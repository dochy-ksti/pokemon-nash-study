# search-turn-max 8 vs 9 / sims 64 vs 128 — 強さと時間 3way

> ⚠️ **訂正 (2026-06-30 後続実験)**: 本実験の「search9 がコヒーレントに弱い (実効の負効果)」
> という結論は **誤り**。3アーム新規 (8/9 も作り直し) で同時期再測定した
> [20260630_2236](20260630_2236_searchmax_7vs8vs9_3way新規再測定.md) では search9 が逆に
> 手法平均トップになり、7/8/9 に有意差なし (全てノイズ帯内)。本実験で search9 を弱いと
> したのは baseline=RC64 流用 + 単発ランの run 間ノイズの偶然で、再現ランを省いた判断ミス。
> sims128 の「強さ向上なし・時間 1.87x」結論は有効。

実施: 2026-06-30 09:16–11:5x JST (probe → funnel A/B → rate, random+crit)

## 目的・仮説

学習教師の探索計算ノブを増やすと方策が強くなるか、時間コストはどれだけかを測る。
`--search-turn-max` (rollout 最大深さ ply) も `--sims` (MCTS シミュレーション数) も
**学習教師の質だけに効く** (funnel eval は policy-only で未使用)。増やすと教師あたりの
生成計算が増える。3 条件で強さ (rate) と時間 (wall/100ep) を比較する。

| 条件 | search-turn-max | sims | 状態 |
|---|---:|---:|---|
| baseline | 8 | 64 | RC64 流用 (RC64_ep110/170/295) |
| A (search9) | **9** | 64 | 新規 funnel (A_s9_ep80/125/195) |
| B (sims128) | 8 | **128** | 新規 funnel (B_sims128_ep110/150/250) |

- baseline 以外は RC64 と **そのノブ以外完全一致**: start=shared_init / depth-skew 2.0 /
  search-min 4 / g64 b512 th128 / minibatch 256 / eps5 / block 50 / warmup 10 / finalists 3 /
  max-added-epochs 1000 / stage 3b / random+crit。
- baseline は新規実行せず [RC64](20260626_1615_学習教師sims_32vs64_random+crit環境_強さAB.md) 流用。
- 段階的方針 (c): まず各 1 ラン、ノイズ帯近辺の微妙な差だけ後で再現ラン。
- driver: `scripts/run_search_sims_3way.sh`。

## 結果

### 強さ (random+crit, rate 9個1プール, n-per-side 512)

| ckpt | 手法 | レート |
|---|---|---:|
| RC64#2 (ep295) | baseline | +10.0 |
| B_sims128#1 (ep150) | sims128 | +8.1 |
| RC64#0 (ep110) | baseline | +6.5 |
| B_sims128#0 (ep110) | sims128 | +4.4 |
| RC64#1 (ep170) | baseline | -0.5 |
| B_sims128#2 (ep250) | sims128 | -0.9 |
| A_s9#2 (ep195) | search9 | -4.7 |
| A_s9#0 (ep80) | search9 | -11.2 |
| A_s9#1 (ep125) | search9 | -11.8 |

**手法平均: baseline = +5.3 / sims128 = +3.9 / search9 = -9.2**

### 時間 (タイミングプローブ: 各100ep, shared_init から逐次, 同一マシン状態)

| 条件 | wall/100ep | baseline 比 |
|---|---:|---:|
| baseline (s8/sims64) | 462s | 1.00x |
| A (s9/sims64) | 500s | **1.08x** |
| B (sims128) | 865s | **1.87x** |

- 参考 (本番 funnel 実 wall, 1000ep 生成 + 選抜): A=1706s / B=3224s
  (B funnel が A の ~1.9x = プローブ比と整合。baseline funnel は流用のため未計測)。
- examples/s 集計は executor 累計カウンタの取り違えで不正確だったため wall/100ep を採用。

## 結論

- **search-turn-max 8→9 (A): 採用しない**。時間 +8% で、強さは **コヒーレントに悪化**
  (手法平均 -9.2, baseline +5.3 比で約 -14 Elo)。**A の finalists 3 個が全体の最下位 3 つ**
  (-4.7/-11.2/-11.8) を占め、ノイズではなく実効の負効果。depth-skew 2.0 と併用すると
  深側 (ply 9) に重みが乗りすぎ、打ち切り地平線の配分が崩れて教師品質が下がった可能性。
- **sims 64→128 (B): 強さ向上なし・時間 1.87x**。手法平均 +3.9 は baseline +5.3 と
  差 1.4 Elo (過去ノイズ帯 ±1.3〜1.5 と同水準) で実質同等。**約 1.9 倍の計算コストに
  見合う強さ向上は無い**。sims 32→64 では +5.6 Elo の効果があった
  ([20260626_1615](20260626_1615_学習教師sims_32vs64_random+crit環境_強さAB.md)) が、
  64→128 では飽和していると見られる (収穫逓減)。
- **結論: baseline (search-max 8 / sims 64) を維持**。search9 は弊害、sims128 はコスト倒れ。
  どちらのノブも現状の強さを伸ばさない。

### 留意

- 1 ラン・random+crit 単一。ただし search9 の負効果は finalists 3 個が揃って最下位で
  コヒーレントなので、ノイズの偶然ではないと判断 (再現ラン優先度は低い)。sims128 の
  「差 1.4 Elo = ノイズ帯」も、向上なしの結論は安全側。
- baseline は RC64 流用 (別シードの既存ラン)。B との比較は run 間ノイズを含むが、差が
  ノイズ帯内なので「向上なし」結論には影響しない。
- 時間は wall/100ep (生成中心)。funnel 全体では eval 固定費も乗るが、ノブのコスト差は
  生成に集中するためプローブ比 (1.08x / 1.87x) がノブ単体のコストを最もよく表す。
