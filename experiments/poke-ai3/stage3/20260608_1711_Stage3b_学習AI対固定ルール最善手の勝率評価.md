# Stage3b 学習AI vs 固定ルール最善手 の勝率評価 (eval-vs-rule)

## 目的

ハンドオフ「次にやること 2」。自己対戦均衡 (交代~0.75) とは別の固定ベースラインで、
学習方策の交代タイミングの妥当性を測る。P1 = 学習 AI (lookahead 探索つき)、
P2 = 固定ルール最善手エージェント (SE可→SE技攻撃 / SE不可→相方へ交代) を対戦させる。

## 実装 (このセッションで追加)

- `poke-ai3/src/rule_agent.rs` (新規): `rule_choice(state, player)`。lookahead/推論を介さず
  真の `BattleState` から決定論着手。SE判定は相手 active 種族→SE技の直接マップ
  (Cloyster→Shock Wave / Goodra→Bulldoze、Python `_se_move_for` と同一)。unit test 2件。
- `poke-ai3/src/game_task.rs`: `run_player` に `rule_based` フラグ。true なら通常ターン・
  強制交代の両方で `rule_choice` を着手 (学習サンプルは積まない)。`run_one_game` に
  `eval_rule_opponent` を追加し P2 をルール側に。評価時は P2 trajectory を root に送らない
  (勝率二重計上の回避)。`game_start`/`async_executor`/pyo3 `lib.rs` に flag を貫通。
- `poke-ai3-python/.../eval_vs_rule.py` (新規) + `eval-vs-rule` エントリポイント。
  P1 (学習側) trajectory の winner と初手対面から、全体+先発対面別の勝率を集計。

## コマンド

```bash
cd poke-ai3-python
uv run eval-vs-rule --checkpoint-path ../data/poke-ai3/stage3b_long.pt \
  --stage 3b --num-games 16 --num-eval-games 512 --sims 64 --sim-concurrency 16 --no-random --no-crit
```

チェックポイント `stage3b_long.pt` (training_step=300)。512 試合。

## わかったこと

1. **学習 AI は固定ルール最善手に対し全体 ~0.48 (ほぼ五分、わずかに負け越し)**。
   自己対戦では「最善手相当」に見えていたが、固定の決定論相手には勝ち越せない。

2. **逆説的な非対称性: 先発有利(SE可)で負け越し(0.405)、先発不利/等倍で勝ち越し(0.551)**。
   型有利で始めたゲームの方が成績が悪い。

3. **解釈: 学習方策は「混合均衡 (Nash mix, 交代~0.75)」のプレイヤーであって、
   特定の固定相手への best-response ではない**。ハンドオフの警告どおり、相手が決定論なら
   最適応手は決定論寄りに変わるのに、学習 AI は均衡混合をそのまま打つため搾取しきれない/
   搾取される。~0.48 は「均衡プレイヤーを固定戦略でつついた」結果として整合的。
   - 有利対面で負け越すのは、有利なのに均衡どおり一定割合で交代してしまい (開幕満タンでは
     交代に勝率優位がない)、決定論相手には手放しになっている可能性。次セッションで対面×着手の
     ドリルダウン (inspect-matchup の win_rates/chosen と突き合わせ) で要検証。

## 注意・次

- この評価は「学習 AI が固定最善手にどれだけ勝てるか/搾取されないか」を見るもので、
  自己対戦均衡 (~0.75 交代が正解) とは評価軸が違う。0.48 は均衡プレイヤーとして妥当で、
  ただちに学習失敗を意味しない。
- 非対称性 (有利対面で弱い) の原因究明が次の論点。固定相手向けの best-response を別に学習させて
  比較する、または有利対面での開幕交代を抑える (value target の max→selection_pi 加重平均への
  変更とも関係) ことで改善するかを A/B する余地。
- 乱数なし (--no-random --no-crit) 条件。crit/roll を入れた頑健性は別途。

## 追記: 「先発有利で負け越す」の究明 (同セッション)

ユーザーが「先発有利で負け越すのは信じがたい」と疑問。追加調査で解明した。

### 追加実装
- `eval_vs_rule.py`: 初手着手別 (攻撃/交代) の勝率内訳を出力。
- `--both-rule` フラグ + Rust `eval_rule_p1`: P1 も固定ルールにして **rule vs rule** の
  構造ベースラインを測れるように。ルール側も初手 1 件だけ state+着手を記録 (対面分類用)。

### 判明したこと (チーム技構成)
`team/poke-ai3/scenario/stage3b_team{1,2}.txt`:
- Team1: Cloyster=Shock Wave, Goodra=Bulldoze
- Team2: Cloyster=Bulldoze, Goodra=Shock Wave

対面ラベルは (自分の技 × 相手種族) だけで、自種族も相手の技も特定しない。
かつ「SE可(有利)」は **自分が相手にSEを撃てる片側指標**で、相手も自分にSEを撃てるかは
無関係。対称ゲームなので「SE可なら相手もSE可」が多く、**SE可 ≠ 盤面有利**。

### rule vs rule (構造ベースライン, n=512)
```
先発有利(SE可)   勝率=0.523  (W138 L126 D0)    初手攻撃100% ・引き分けゼロ
先発不利/等倍    勝率=0.258  (W64  L63  D121)  初手交代100% ・引き分け約半数
```
- 有利(SE可)=両者攻撃→決着、構造的にほぼ五分 (0.52)。純有利は相殺されほぼ無い。
- 不利/等倍=両者交代し合い MAX_TURNS 引き分けが 121/248 (≈49%)。決着分は 64:63 で五分。
  勝率 0.258 は引き分けを非勝利として数える指標の見かけ。

### 学習AI vs rule との比較 (初手着手別, run2)
```
SW_v_Cl(有利)  全体0.421  初手攻撃0.324  初手交代0.534
BD_v_Go(有利)  全体0.369  初手攻撃0.354  初手交代0.435
```
| 先発 | 学習AI | rule基準 | 差 |
|---|---|---|---|
| 有利(SE可) | 0.395 | 0.523 | −0.13 (取りこぼし) |
| 不利/等倍   | 0.545 | 0.258 | +0.29 (引き分け回避) |

### 結論 (信じがたさの解消)
1. **「先発有利で負け越す」は錯覚気味**: SE可は構造的にほぼ五分 (相殺) で元々大きな有利でない。
2. **学習AIの真の弱点**: 有利対面の攻撃レースで rule なら 0.52 のところ 0.395 (−0.13)。
   決定論的に殴る相手に後続ターンで不要な交代を混ぜ、勝てるレースを落とす
   (= 均衡混合のまま固定相手に best-response できない / 押し切りが弱い)。
3. **学習AIの強み**: lookahead で交代スタックの引き分けを破り、不利対面の引き分けを勝ちに変える
   (0.545, 引き分けほぼ 0)。
4. 全体 0.47 は「不利で引き分け拾い (+)」と「有利で攻撃レース取りこぼし (−)」の差し引き。
   学習失敗ではないが、**有利時の詰め (押し切り)** が固定相手相手に弱いのが明確化。

### 次の論点
- 有利対面での取りこぼし改善: 固定相手向け best-response の別学習、または有利時の
  不要交代を抑える (value target max→selection_pi 加重平均の検討とも関係) A/B。
- 引き分け (両者交代ループ) の扱い: 学習は自己対戦で交代均衡を学ぶが、固定相手では
  詰めが要る。報酬・探索での「押し切り」インセンティブ設計の検討余地。

## 追記2: pick_config の同チーム対戦バグ発見・修正 (同セッション)

ユーザー指摘「お互いが SE を撃てる状況は存在しないようチーム編成されている。SE可なら常に有利」
を検証。チーム設計 (Team1=自種族にSE / Team2=相手種族にSE) により **クロスチーム
(Team1 vs Team2) なら必ず片側だけ SE 可** = SE可は厳密に有利、は正しい。

ところが `pick_config` (game_task.rs) は P1/P2 のチームを独立に選んでおり、全16通り中
**8通りが同チーム対戦** (T1vsT1 / T2vsT2) だった。同チームは:
- 両者SE (ミラー) 4通り / 双方非SE 4通り
これが eval の「有利」バケツにミラーを混入させ、rule vs rule の不利バケツに 49% の
交代ループ引き分けを生んでいた (= 解釈を濁していた正体)。

**修正**: pick_config を常に Team1 vs Team2 (クロスチーム8通り: swap × active1 × active2) に。
不変条件テスト `stage3b_pick_config_is_always_cross_team` 追加。

含意: **既存 stage3b_long.pt は汚染前分布 (半分が縮退局面) で学習済み**。本来の修正は再学習。
当面は修正後のクリーンな対面で現チェックポイントを測り直し、真の有利を活かせるか確認する。
修正後の対面マッピング (全てクロスチーム=片側SE):
- 有利 (P1がSE可・相手SE不可): SW_v_Cl, BD_v_Go
- 不利 (相手がP1にSE可):       SW_v_Go, BD_v_Cl

### 修正後の再評価 (現チェックポイント stage3b_long.pt, n=512)
| P1 方策 | 全体 | 有利 | 不利 | 引分 |
|---|---|---|---|---|
| 固定ルール(best-move) | 0.500 | 0.502 | 0.498 | 0 |
| 学習AI(lookahead)     | 0.188 | 0.215 | 0.160 | 0 |

- **rule vs rule クロスチームは完全に 0.500** (全バケツ ~0.5、引き分けゼロ)。最善手同士なら
  ゲームは完全公平。**「先発有利(SE可)」でも 0.502** = 純有利はほぼゼロ。相手が SE を撃てる
  相方へ即ピボットして有利を取り返すため。SE可は局所的優位だが最適応手で ~0.5 に中和される。
- **同じ P1 席で best-move ルール 0.50 に対し学習AI 0.19 (−0.31)**。ルールは公平 (0.5) なので、
  これは学習AIが単純な決定論戦略に大きく搾取されていることを意味する。均衡混合を固定相手に
  そのまま打ち、ピボット合戦に追従できていない。
- **但し書き**: stage3b_long.pt はバグ修正前の汚染分布で学習済み。0.188 は汚染学習モデルを
  クリーン対面で測った値で、真の実力とは言い切れない。正しい評価には修正後分布での再学習が要る。
