# 観測パック転送 + エンコード offload でスループット約 3 割改善

## 背景

CUDA Graphs 化後もメインスレッド (推論ポンプ) が 100% で律速。バッチサイズは
実測で平均 235/256 と健全 (20260612 バッチ分布プローブ) で、残る固定費は
ラウンドあたり ~1.26ms の Python/FFI/転送経路だった。

## 実装

1. **パック転送**: 観測 21 配列を行優先の 3 行列 (`packed_i64` (B,20) /
   `packed_f32` (B,27) / `legal_action_mask` (B,ACTION_DIM)) に連結。
   列レイアウトは `poke_ai3::packed` (Rust) が唯一の正で、Python へ
   `ENCODED_PACKED_LAYOUT` (JSON) としてエクスポート。H2D 転送は 21 回 → 3 回、
   CUDA Graph の静的バッファへ直接 `copy_`。列→フィールド分解 (`packed_views`)
   はキャプチャ時にグラフへ焼き込まれ replay コストゼロ。
2. **エンコード offload**: `encode_batch` + パックを Python メインスレッド
   (`recv_observations`) から RootTask (tokio ワーカー) へ移動。recv は出来上がった
   バッファを numpy に包むだけ (実測 1.26ms → 0.034ms)。
3. **sleep 50ms 罠の修正**: offload によりバッチ到着がエンコード分遅れ、メイン
   ループが `sleep(0.05)` に入って全ゲームが 50ms 停止する準安定スローダウンが
   顕在化 (30ep が 100s → 600s 超)。`--sleep-seconds` デフォルトを 0.05 → 0.001 に変更。

診断系経路 (`encode_observation_states`: play_gui / supervised / inspect) は
従来のフィールド別 dict のまま。golden テスト・全 Rust テスト通過。

## 計測 (30ep, num_games 32, W=16, sims 64, depth 6-12, seed 20260612)

| 構成 | wall | CPU% | samples/s |
|---|---|---|---|
| baseline (e4c8aed, sleep 50ms) | 102.2s | 657% | 182.4k |
| baseline (sleep 1ms) | 99.0s | 646% | 174.7k |
| パックのみ (main thread, 50ms) | 104.8s | 670% | 179.5k |
| パック+offload (sleep 50ms) | **626s** | 147% | ~30k (準安定スローダウン) |
| パック+offload (sleep 1ms) | **78.9s** | **787%** | **235.1k (+29%)** |

フェーズ別 (offload 後): recv 0.034ms / infer 0.492ms / send 0.233ms per round。

## 計測上の教訓

- `maturin develop` は train-loop 起動内で走るため、**プロセスにロード済みの旧 .so
  は差し替わらない**。Rust 変更直後の 1 回目の実行は旧バイナリで動く。ブランチ/
  stash 切替後は明示的に `uv run python -m maturin develop` してから計測すること。
  (これにより offload 版の初回計測が「速い」と誤認された)

## 次の候補

- メインスレッドの残コスト: infer 0.49ms (グラフ入力 copy_ 3 回 + replay + DtoH
  同期) と send 0.23ms (256 行のルーティング: 行ごとの channel send)。send の
  ルーティングを Rust 側 (tokio) へ offload するのが次の一手。
- 飽和点が動いたので num_games / sim-concurrency のスケーリング再計測も価値あり。

## 追記: sleep_seconds=0 の計測

| sleep | wall (30ep) | CPU% | samples/s |
|---|---|---|---|
| 1ms | 78.9s | 787% | 235.1k |
| 0 (ビジーポーリング) | 73.9s | 820% | 247.0k (+5%) |

`time.sleep(0)` は GIL を解放して即戻るため待ちの谷が消える。暇な期間もメイン
スレッドが 1 コア回り続けるが、学習専用マシンでは実害なし。既定を 0 に変更した。
