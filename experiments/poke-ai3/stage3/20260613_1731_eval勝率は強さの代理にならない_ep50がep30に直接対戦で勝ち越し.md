# eval 勝率は「強さ」の代理にならない — 学習が進んで eval 勝率が下がった ep50 が、eval 勝率の高い ep30 に直接対戦で勝ち越す

## 目的

rule AI (P2, weakness_move_rate=100% の決定論方策) との eval 勝率は、それを最大限スポイル
できる方策を高く評価する。しかし真の最善は混合戦略ナッシュ均衡で、weakness_move_rate は
状況に応じた適度な値が強い。よって「eval 勝率が高い = 実際に強い」とは限らない。
seed1 の学習曲線で eval 勝率が ep30 でピーク (0.789) → ep50 で低下 (0.719) した現象が、
「過学習による劣化」なのか「rule AI 特化から均衡寄りへ移動した結果の見かけの低下」なのかを、
rule AI を介さず ep30 と ep50 を**直接対戦**させて切り分ける。

## 方法

新ツール `eval_ckpt_vs_ckpt` を追加 (poke-ai3-python, Python のみ・Rust 変更なし)。

- self-play executor (両側 NN 推論, eval_rule_opponent=False) を学習なしで回す。
- 観測バッチを `obs["player"]` (0=P1, 1=P2) で分割し、P1→checkpoint A / P2→checkpoint B に
  ルーティング (`infer_step_split`)。両 agent をフルバッチに通し player マスクで採用行を選ぶ。
- trajectory の winner を [A勝, B勝, 引分] で集計。
- 先発 side 有利を相殺するため A/B の P1/P2 を入れ替えた 2 回を実施。
- **注意**: checkpoint は絶対パスで渡す。`data/` はリポジトリ root にあり poke-ai3-python から
  相対 `data/...` は解決できず、無言で random init (step=0) になる事故があった (本実験で発見)。

## コマンド

```bash
cd poke-ai3-python
# ep30 を P1, ep50 を P2
uv run python -m poke_ai3_train.eval_ckpt_vs_ckpt \
  --checkpoint-a /home/dochy/pokemon_ai_proj/data/poke-ai3/ckpt_curve_ep30.pt \
  --checkpoint-b /home/dochy/pokemon_ai_proj/data/poke-ai3/ckpt_curve_ep50.pt \
  --num-games 16 --num-eval-games 256 --sim-concurrency 16 --sims 64 \
  --search-turn-min 6 --search-turn-max 12 --no-random --no-crit --stage 3b
# 入れ替え (ep50 を P1)
uv run python -m poke_ai3_train.eval_ckpt_vs_ckpt \
  --checkpoint-a .../ckpt_curve_ep50.pt --checkpoint-b .../ckpt_curve_ep30.pt ...
```

両 checkpoint とも seed1 (ckpt_curve)、hidden128。rule AI eval 勝率は ep30=0.789 / ep50=0.719。

## 結果 (head-to-head, n=256/側)

| 配置 | ep50 視点の勝率 |
|---|---:|
| ep30=P1, ep50=P2 | ep50 = **0.641** (164/256) |
| ep50=P1, ep30=P2 | ep50 = **0.664** (170/256) |

両 side で一致して **ep50 が ep30 に勝ち越し** (平均 ~0.65)。side を入れ替えても結論が
変わらないので先発有利のアーティファクトではない。

## 発見

- **eval 勝率の順位 (ep30 > ep50) は、実際の強さの順位 (ep50 > ep30) と逆転している。**
  rule AI との eval 勝率は強さの信頼できる代理ではない、というユーザ仮説を直接対戦で裏付けた。
- ep30→ep50 の eval 勝率低下 0.789→0.719 は「過学習による劣化」ではなく、rule AI 特化方策
  から**より均衡寄りの方策へ移動**した結果の見かけの低下と解釈できる (より強い相手=ep30 自身に
  対しては勝ち越す)。
- 「学習が進むほど eval 勝率が下がりうる」という非単調性は、収束していないのではなく、
  **評価指標が学習目標 (自己対戦均衡) とずれている**ことの表れの可能性が高い。

## 結論と次の一手

- 学習の進捗・モデル比較を rule AI eval 勝率**だけ**で判断するのは危険。直接対戦リーグ
  (Elo / 総当たり) を主指標に格上げすることを検討すべき。
- nash.rs の「平均の半分以下を最小に落とす」処理の是非も、rule AI eval ではなく
  崖あり/なしモデルの直接対戦で判定するのが筋。
- 残課題: 冒頭 ~15ep 全敗 (交代崩壊仮説) は本実験とは独立。trajectory dump で別途切り分け。
- 追加候補: cross-seed (seed1 ep50 vs seed2 ep50) で eval 勝率順位が直接対戦と一致するかも検証。
