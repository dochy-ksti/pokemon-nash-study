# stage 3b 300 epoch: long_xteam 対等条件 vs 深設定 (sims 512) の比較

実施: 2026-06-12 15:04 JST

## 目的

Rust エンコード化 (bbe5ec2) 後の経路で stage 3b を 300 epoch 学習し、
(1) 現行基準 `stage3b_long_xteam.pt` (0.564) と対等条件での品質比較、
(2) 深い探索設定 (sims 512 / W 256 / batch 8192 / turn 4-8) での品質と時間を比較する。

## コマンド

対等条件 (`stage3b_xteam_eq300.pt`):

```bash
cd poke-ai3-python
uv run train-loop --num-games 32 --sim-concurrency 16 --sims 64 \
  --search-turn-min 6 --search-turn-max 12 --no-random --no-crit --stage 3b \
  --max-epochs 300 --checkpoint-path ../data/poke-ai3/stage3b_xteam_eq300.pt
```

深設定 (`stage3b_fast300.pt`):

```bash
uv run train-loop --num-games 32 --sim-concurrency 256 --sims 512 \
  --search-turn-min 4 --search-turn-max 8 --no-random --no-crit --stage 3b \
  --max-epochs 300 --max-batch-size 8192 --checkpoint-path ../data/poke-ai3/stage3b_fast300.pt
```

評価はいずれも基準と同条件:

```bash
uv run eval-vs-rule --checkpoint-path <ckpt> --stage 3b \
  --num-games 16 --num-eval-games 512 --sims 64 --sim-concurrency 16 --no-random --no-crit
```

ログ: `logs_stage3b_xteam_eq300.log` / `logs_stage3b_fast300.log` / `logs_eval_*.log`

## 結果 (eval-vs-rule, n=512)

| モデル | 学習条件 | 学習時間 | 全体 | 有利(SE可) | 不利/等倍 |
|---|---|---|---|---|---|
| stage3b_long_xteam (基準, JSON 経路時代) | sims 64, W 16, turn 6-12, 300ep | (旧経路, 数時間規模) | 0.564 | 0.576 | 0.553 |
| **stage3b_xteam_eq300** (対等条件) | sims 64, W 16, turn 6-12, 300ep | **1:06:24** | **0.754** | 0.734 | 0.773 |
| stage3b_fast300 (深設定) | sims 512, W 256, batch 8192, turn 4-8, 300ep | 1:14:55 | 0.633 | 0.668 | 0.598 |
| (参考) stage3b_fast30 | 深設定 30ep | 0:03:53 | 0.174 | 0.224 | 0.125 |
| rule vs rule 基準 | - | - | 0.500 | 0.502 | 0.498 |

## 考察

- 対等条件の再学習が基準 0.564 → **0.754** と大幅に上回った。コード上の学習条件は
  long_xteam と同一のはずなので、差分は (a) seed / 学習の分散、(b) bbe5ec2 までの
  リファクタリング (d3ae347 の Embedding+pointer 化以降の挙動差) のどちらか。
  いずれにせよ新経路で品質の退行はない (むしろ向上)。
- 深設定 (sims 512 / turn 4-8) は同 300ep で 0.633 と対等条件より低い。
  sims を増やしても search_turn 4-8 の浅い読みでは教師の質が上がらない可能性。
  時間も 1:14:55 と対等条件より長く、この設定は epoch あたりの探索量が大きい割に
  得るものが少ない。
- 30ep (0.174) → 300ep (0.633) で学習量が支配的なことも確認。

## 結論 / 次の候補

- 新経路での 300ep 学習は約 1 時間で回り、対等条件で基準を大きく上回る 0.754。
- 探索パラメータは「sims 64 / turn 6-12」が現状ベスト。深設定を活かすなら
  sims よりも search_turn を伸ばす方向の試行が候補。
- 分散切り分けのため対等条件を別 seed でもう 1 本回す価値あり。
