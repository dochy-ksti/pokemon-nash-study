# Stage3b 対面別ポリシー診断 — 交代率75%頭打ちの正体

## 目的

`stage3b_long.pt`(300ep)の交代率が ~75% で頭打ちになった原因を、
(active の技 × 相手 active 種族) の4対面ごとに「モデル生 policy / 教師 target_pi /
selection_pi / 実際の着手頻度」と、モデル交代確率の HP 別内訳で調べる。

仮説(ユーザー): Shock Wave は Goodra に半減だが Bulldoze は Cloyster に等倍。この
非対称性が交代率を 90% でなく 75% に抑えているのではないか。有利対面でも
Shock Wave 持ち→Bulldoze 持ちの交代が有効かもしれない。

## コマンド

学習せず(チェックポイント非上書き)、推論のみで trajectory を集めて集計する新ツール
`inspect-matchup` を追加して実行:

```bash
cd poke-ai3-python
uv run inspect-matchup \
  --checkpoint-path data/poke-ai3/stage3b_long.pt \
  --stage 3b --num-games 8 --num-batches 4 \
  --sims 64 --sim-concurrency 8 --no-random --no-crit
```

training_step=300 / total items=726。

## 結論

1. **学習ギャップは無い**。全対面で model ≈ target_pi ≈ selection ≈ chosen。
   75% は lookahead 均衡をモデルが忠実に再現した結果であり、学習不足ではない。

2. **交代判断を決めるのは「active が SE を通せるか」だけ**で、半減/等倍の区別は無関係。
   - SE 対面 (SW_v_Cl, BD_v_Go): 交代 ~0.21–0.28(=攻撃を選ぶ)
   - 非SE 対面 (SW_v_Go=半減, BD_v_Cl=等倍): 交代 ~0.74–0.76(=SE で殴れる相方へ交代)
   等倍の BD_v_Cl(0.758)は半減の SW_v_Go(0.739)とほぼ同じ交代率。
   → 仮説の非対称性は交代率を下げていない。Bulldoze が等倍でも、相方の SW が
     Cloyster に SE で通るため「交代した方が良い」点は半減ケースと変わらない。

3. **「有利対面でも交代有効」は否定**。SE 対面は攻撃 ~73–79%。有利なら殴る。

4. **75% 頭打ちの正体は (a) 混合ナッシュ均衡 + (b) HP 状況の混合**。
   非SE対面の交代率は一様 75% ではなく HP で二峰性。開幕(両者満タン my_hi/opp_hi)は
   交代が低く(SW_v_Go 0.33 / BD_v_Cl 0.45, ここの n が大きい)、被弾後は 0.8–1.0 に上がる。
   集計 0.75 はこの混合の平均で、交代率が 75% で「天井」に張り付いているわけではない。
   開幕満タン対面で交代が低いのが平均を押し下げている。
