# Stage3a win_rate 弱体化の再現とマッチアップ別分析

JST: 2026-06-05 15:30

## 目的

前実験 (20260605_1523) で観測した、学習後半の win_rate 低下が再現するかを別 run で
確認する。併せて **マッチアップ別勝率** (P1視点 my_v_opp の4カード) を新たに計測し、
弱体化が「特定カードの戦略的劣化」か「全カード一様」かを切り分ける。

## 変更

- `supervised.py` / `agent.py` / `train_loop.py`: 先発対面ごとのマッチアップ別勝率
  (Cl_v_Cl / Cl_v_Go / Go_v_Cl / Go_v_Go) をログ出力に追加。1v1 は対面が試合中一定なので
  各 trajectory の先頭 state で分類。

## コマンド

```bash
uv run train-loop --num-games 32 --sim-concurrency 16 --sims 64 \
  --search-turn-min 4 --search-turn-max 8 --no-random --no-crit --stage 3a \
  --max-epochs 40 --chunk-threshold 512 \
  --checkpoint-path ../data/poke-ai3/stage3a_run2.pt   # 新規 (前 run の重みは読まない)
```
ログ: `data/poke-ai3/stage3a_train2.log`

## 結果

- **win_rate 弱体化は再現**。correct_action_rate が 1.0 に到達 (~epoch 19) した直後から
  win_rate が 0.0〜0.19 へ崩落し、**epoch 40 まで回復しない**。
- マッチアップ別: 崩落は **4カードすべてで一様**。対称な鏡像対面 **Cl_v_Cl・Go_v_Go も
  0.0 付近**まで低下 (収束後)。

| 局面 | correct_action | win_rate | 鏡像 (Cl_v_Cl, Go_v_Go) |
|---|---|---|---|
| 収束前 (epoch 1–18) | ~0.5 振動 | ~0.3–0.75 | ~0.5 付近で振動 |
| 収束後 (epoch 19–40) | 1.0000 | 0.0–0.19 | **~0.0** |

## 分析・根本原因

- ターン上限が無く (`while !state.is_done()`)、技は必ずダメージを与えるため **引き分けは
  発生しない**。よって win_rate≈0 は引き分けではなく **P1 のほぼ全敗**。
- 両者は同一 net の自己対戦。P1/P2 間の唯一の非対称は **先手決定** (`local_showdown.rs`)。
  先手は seed 由来 `tiebreak` の LCG パリティで決まり、LCG は奇数係数・奇数加算なので
  **パリティが毎ターン反転**。決定論的に収束すると、決着ターンの先手が seed パリティだけで
  決まり、per-game seed のパリティ偏りで P1 が系統的に不利になる。
- **結論: win_rate 弱体化は決定論的自己対戦における先手アーティファクトであり、戦略的
  能力の劣化ではない。** 弱点突き (correct_action=1.0) は崩落中も維持されている。
  鏡像カードまで一様に 0 へ落ちることが、戦略劣化ではないことの決定的証拠。

## 推奨する対応 (未実装)

1. **勝率指標の対称化**: 現在 win_rate は P1 trajectory のみ集計。P1/P2 両視点を平均すれば
   先手アーティファクトが相殺され ≈0.5 になる (不偏推定)。`build_examples` を両者集計に変更。
2. **先手 RNG の独立化**: seed パリティに連動しない独立コインで先手を引く。速度学習は後段
   マイルストーンなので、それまでは先手をゲームごとに公平にランダム化する。
3. それまでは win_rate を収束判定指標として使わない (correct_action_rate / マッチアップ別を使う)。
