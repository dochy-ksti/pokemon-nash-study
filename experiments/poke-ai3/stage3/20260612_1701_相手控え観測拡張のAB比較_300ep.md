# 相手控え観測拡張の A/B 比較 (stage 3b, 300 epoch, 新旧各 1 ラン)

実施: 2026-06-12 17:01 JST 開始

## 目的

相手控え観測拡張 (docs/poke-ai3/stage3/20260612_1631) の効果検証。
ベースラインは合意 8/9 のとおり「新コード + `--mask-opp-obs`」(アーキテクチャ
完全同一・入力情報のみ差分)。新旧各 1 ランの軽量比較 (合意 11) であり、
1 seed では差がノイズか効果か区別できないため、結果は「スモーク + 方向感の確認」
として解釈する。

## 仮説 (実施前に記載)

- 相手控えの種族・HP・技と相手 active の技が見えることで、詰め筋・交代読みの
  学習が可能になり、対固定ルール勝率 (eval-vs-rule) が改善する方向に出る。
- value 較正 (20260608_1641 の歪み) の改善は AI 側の予想であり、ユーザーは
  懐疑的 (合意 11)。本実験では軽く確認する程度に留める。
- 学習が混乱する兆候が出た場合の第一容疑者は「敵味方の区別を位置埋め込みのみに
  任せた」点 (合意 10)。

## 条件

対等条件 (20260612_1504 の現状ベスト): sims 64 / W 16 / turn 6-12 / 300ep。
両アームとも同一 `--battle-seed 20260612` で対戦生成を揃える
(GPU 並列タイミングによりビット単位再現はないが、条件は揃う)。

## コマンド

新観測 (`stage3b_oppobs300.pt`):

```bash
cd poke-ai3-python
uv run train-loop --num-games 32 --sim-concurrency 16 --sims 64 \
  --search-turn-min 6 --search-turn-max 12 --no-random --no-crit --stage 3b \
  --max-epochs 300 --battle-seed 20260612 \
  --checkpoint-path ../data/poke-ai3/stage3b_oppobs300.pt
```

旧観測相当ベースライン (`stage3b_maskobs300.pt`): 上記 + `--mask-opp-obs`、
checkpoint を `../data/poke-ai3/stage3b_maskobs300.pt` に変更。

評価 (両アーム、基準と同条件 + ベースライン側のみ `--mask-opp-obs`):

```bash
uv run eval-vs-rule --checkpoint-path <ckpt> --stage 3b \
  --num-games 16 --num-eval-games 512 --sims 64 --sim-concurrency 16 --no-random --no-crit
```

ログ: `logs_stage3b_oppobs300.log` / `logs_stage3b_maskobs300.log` /
`logs_eval_stage3b_oppobs300.log` / `logs_eval_stage3b_maskobs300.log`

## 結果 (eval-vs-rule, n=512)

| アーム | 全体 | 有利(SE可) | 不利/等倍 |
|---|---|---|---|
| **新観測 (stage3b_oppobs300)** | **0.635** | 0.663 | 0.607 |
| 旧観測相当 (stage3b_maskobs300) | 0.559 | 0.584 | 0.533 |
| (参考) rule vs rule 基準 | 0.500 | 0.502 | 0.498 |
| (参考) 20260612_1504 対等条件 (旧コード・別 seed) | 0.754 | 0.734 | 0.773 |

先発対面別 (全体勝率、新観測 / 旧観測相当):

| 対面 | 新観測 | 旧観測相当 |
|---|---|---|
| SW_v_Cl (有利) | 0.669 | 0.567 |
| SW_v_Go (不利) | 0.594 | 0.481 |
| BD_v_Cl (等倍) | 0.620 | 0.586 |
| BD_v_Go (有利) | 0.656 | 0.602 |

- 両アームとも 300 epoch 完走、学習はそれぞれ約 1 時間 (対等条件の従来実績どおり)。
  マスク有効化はログ先頭の `mask_opp_obs: enabled` で確認。
- 学習自体はどちらも安定 (loss 発散・交代率の崩壊なし)。合意 10 で警戒した
  「位置埋め込みのみの敵味方区別」起因の混乱は見られなかった。

## 考察

- 新観測 0.635 vs 旧観測相当 0.559 (+0.076)。4 対面すべてで新観測が上回り、
  方向は仮説どおり「観測拡張は有効」。ただし各 1 ラン (合意 11) であり、
  20260612_1504 で見たとおり seed 分散はこの差分と同程度になり得るため、
  確定的な結論ではなくスモーク + 方向感の確認。
- 差が最も大きいのは不利対面 SW_v_Go (+0.113)。相手控えの HP が見えることで
  交代後の詰め筋を読みやすくなる、という仮説と整合する出方ではある。
- 参考値の旧コード 0.754 とは直接比較できない (HP 注入整理 + 別 seed の交絡が
  あるため、合意 8 でベースラインを新コード側に置いた)。新コードの絶対水準が
  気になる場合は同条件の追加 seed で分散を見るのが次の一手。

## 結論

- 観測拡張ありが全体・全対面で上回り、効果の方向はプラス。1 seed の軽量比較
  なので、効果量を確定したい場合のみ追加 seed を回す (現時点では必須としない)。
- value 較正の精査 (20260608_1641 の再実施) は未実施。必要になったら別途。
