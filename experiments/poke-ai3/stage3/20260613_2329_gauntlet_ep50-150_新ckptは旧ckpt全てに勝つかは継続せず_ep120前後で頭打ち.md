# gauntlet ep50→150: 「新 checkpoint は過去の強い checkpoint 全てに勝つ」は継続しない — ep120 前後で頭打ち、以降は横移動

## 目的

連続学習で「より多く学習した checkpoint は過去のどの checkpoint にも勝ち越す」(強さの単調・推移的
向上) が **どこまで成り立ち続けるか** を、5ep 刻みの gauntlet で ep150 まで追う。成り立てば
「直近 ckpt = 最強」と扱え、直接対戦リーグを学習進捗の指標にできる。

前提 (前実験):
- rule AI eval 勝率は強さの代理にならない (eval 0.789 の ep30 が eval 0.719 の ep50 に直接対戦で負ける)。
- 近接世代でじゃんけん性 (準・非推移性) の兆候。
本実験はそれを ep150 まで延長して定量化したもの。

## 方法

orchestrator `scripts/gauntlet.py`。各ラウンド: 5ep 追加学習 → 新スナップショットを「直近 5 世代」
プール全員と直接対戦 (eval_ckpt_vs_ckpt, 先発有利相殺で両 side, 各対戦 n=256=128×2)。
新スナップショットをプールに追加し直近 5 に絞る。新 ckpt が全員に勝率>0.5 なら「全勝」。

- ep50→70: 1 回目の run (4 ラウンド, 別記録 20260613_1905)。
- ep70→150: 本 run (16 ラウンド)。開始 ckpt = ep70 の作業 checkpoint、プールは ep70 終了時点の
  [curve_ep50, r1_ep55, r1_ep60, r1_ep65, r1_ep70] を継承。
- hidden128, stage3b, no-random/no-crit, sims64, search 6-12。学習は num-games 32。

## コマンド

```bash
cd poke-ai3-python
D=../data/poke-ai3
uv run python scripts/gauntlet.py --start $D/gauntlet/r1_work.pt \
  --rounds 16 --epochs-per-round 5 --pool-size 5 --n-per-side 128 --tag r2 \
  --seed-pool $D/ckpt_curve_ep50.pt $D/gauntlet/r1_ep55.pt $D/gauntlet/r1_ep60.pt \
             $D/gauntlet/r1_ep65.pt $D/gauntlet/r1_ep70.pt
```

## 発見

### 1. 仮説は継続的には成り立たない

「新 ckpt が直近 5 世代全員に勝つ」は散発的にしか成立しない。大半の取りこぼしは誤差レベル
(ちょうど 128-128 の .500、または 95%CI≈±0.06 内) だが、**誤差で説明できない実後退**も存在:
ep85→ep60 .414、ep115→ep90 .453、ep145→ep125 .453。新しい方が確実に強いとは限らない。

### 2. ep120〜130 が強いプラトー (局所ピーク) を形成し、以降が超えられない

ep120/125/130 は珍しく 3 連続で全勝。しかし続く ep135/145/150 はこのクラスタを明確に超えられず、
**ep150 vs ep125 = .492** (25ep 新しい ep150 が ep125 に勝てない)。ep120 前後で学習が頭打ちになり、
以降は強くなるというより方策空間を横移動している描像。

### 3. 遠い過去には概ね勝つ (中期向上は ep120 まで実在)

各ラウンドで 5 世代前 (=25ep 前) への勝率は概ね .55〜.64。ep120 あたりまでは中期スパンの向上は
実在する。崩れるのはそこから先。

## 結論と次の一手

- **「直近 ckpt が常に最強」は成り立たない。** 学習進捗・モデル選択は単一の直近 ckpt でなく
  複数世代の総当たり (Elo/リーグ) で測るべき。本実験 (80ep, 計 80 対戦) が定量的裏付け。
- **ep120 前後で実質的に学習が頭打ち。** これ以上 epoch を回しても (現設定では) 強くならず横移動。
  改善には epoch 追加でなく、学習設定・モデル・nash 累積アルゴリズム側の変更が必要。
- 当初の論点 (nash.rs の「平均の半分以下を最小に落とす」崖の是非) は、まさにこの「頭打ち後に
  横移動」を改善できるかで評価すべき。崖あり/なしで ep0→120 を学習し、到達プラトー水準を
  総当たり Elo で比較するのが筋。
- 近接世代の非推移性 (じゃんけん性) をさらに詰めるなら n=1024・全 ckpt 総当たり行列で
  非推移性検定。

## 追記: ep150→200 と弱体化検証 (固定アンカー方式)

「ep140 以降弱くなっているのでは」を確かめるため ep200 まで延長。移動プール (gauntlet) は
基準がドリフトするので、固定アンカー ep105 / ep130 に対し各世代を両 side 対戦 (各 n=256) させ、
被験体視点の勝率トレンドを取る (`scripts/anchor_sweep.py` 新規)。ep150→200 は train-loop
--snapshot-every 5 で追加学習 (r2_work_ep155..200)。

### 結果 (被験体視点の勝率)

| 世代 | vs ep105 | vs ep130 | 平均(強さ指標, n=512, SE≈0.022) |
|---|---|---|---|
| ep110 | .516 | .430 | .473 |
| ep120 | **.594** | .492 | .543 |
| ep140 | .492 | **.625** | **.559** |
| ep150 | .453 | .578 | .516 |
| ep160 | .484 | .492 | .488 |
| ep170 | .500 | .477 | .489 |
| ep180 | .500 | .414 | .457 |
| ep190 | .430 | .531 | .481 |
| ep200 | .484 | .492 | .488 |

### 判定: 弱体化は「本物だが一段沈んで横ばい」(暴落ではない)

- **ピークは ep120〜140** (強さ指標 ep140=.559 が最大)。**ep160 以降は ~0.46〜0.49 に落ちて横ばい**。
  ピーク (.559) とプラトー (~.48) の差 ≈0.08 ≒ 3〜4 SE で **有意な低下**。「ep140 から弱化」は正しい方向。
- **ただし下がり続けてはいない。** ep160→200 の変動 (.457〜.489) は 1〜1.5 SE 内のノイズで、
  ep200 が ep160 より悪いわけではない。ピーク→一段下のプラトーへ沈んで固定しただけ。暴走的崩壊は ep200 まで無し。
- **ep160 以降はどちらのアンカーにも勝ち越せず (≤0.5)**。ep120〜140 の水準が天井で、+45〜80ep でも超えられない。

→ 前述の「ep120 前後で頭打ち」に、「その後わずかに (有意に) 劣化して一段下で横ばい固定」が加わった。
強さスカラー単調増加でないことに加え、**過学習的な軽い後退**が起きている可能性。学習継続では改善せず、
設定・モデル・nash 累積アルゴリズム側の変更が必要、という結論を補強。
