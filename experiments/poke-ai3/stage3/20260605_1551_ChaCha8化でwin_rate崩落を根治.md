# ChaCha8 化で win_rate 崩落を根治

JST: 2026-06-05 15:51

## 背景

Stage3a 学習で、correct_action_rate が 1.0 に収束した直後から win_rate が 0 付近へ崩落
する現象を観測 (20260605_1523, 1530)。原因は、決定論的自己対戦における速度タイの退化:
- `local_showdown.rs` の `tiebreak` は LCG で更新され、その **最下位ビット**で先手を決定。
- LCG の bit0 は周期 2 で毎ターン反転。さらに seed は 4 連続整数の XOR で初期パリティが
  常に 0。結果、**全ゲームで先手順が `P2,P1,P2,…` に固定**され、収束後に P1 がほぼ全敗。

## 変更 (LCG 全廃 → ChaCha8 統一)

- 新規 `poke-env-rust/src/battle_chacha.rs`: `rand_chacha::ChaCha8Rng` ベースの `BattleChaCha`。
  ダメージ乱数・急所・速度タイ (`first_player`)・rollout 深さ (`below`)・方策サンプリング
  (`unit`) を全ビット良質な 1 本のストリームから供給。seed から完全再現可能。
- `local_showdown.rs`: `LocalRng`(LCG) と自前 `tiebreak` LCG を撤去し `BattleChaCha` に。
  先手は毎ターン `rng.first_player()`(公平コイン)。
- `lookahead.rs` / 旧 `rollout_rng.rs`: `LcgRng` を撤去し `BattleChaCha` に統一。先手
  `rng.first_player()`、深さ `rng.below(span)`。
- `poke-env-rust/Cargo.toml`: `rand` / `rand_chacha` を追加。
- 単体テスト: 同 seed で先手列が一致 (再現性)、seed 横断で turn1 P1 率 ≈0.5 (公平性)。

## 検証

```bash
uv run train-loop --num-games 32 --sim-concurrency 16 --sims 64 \
  --search-turn-min 4 --search-turn-max 8 --no-random --no-crit --stage 3a \
  --max-epochs 25 --chunk-threshold 512 --checkpoint-path ../data/poke-ai3/stage3a_chacha.pt
```
ログ: `data/poke-ai3/stage3a_chacha.log`

| | correct_action 収束後の win_rate | 鏡像 (Cl_v_Cl/Go_v_Go) |
|---|---|---|
| 修正前 (LCG) | 0.0–0.19 に崩落・固定 | ~0.0 固定 |
| 修正後 (ChaCha8) | **≈0.5 中心** (epoch10–25 平均 ≈0.52, 範囲 0.25–0.69) | ~0.5 付近で正常変動 |

- correct_action_rate は epoch10 以降 1.0 を維持 (弱点突きは健在)。
- win_rate の 0 崩落は消失。残る揺れは n=16/epoch のサンプリングノイズ (±0.12 程度)。
- `cargo test --workspace` 全通過。

## 結論

win_rate 崩落は LCG 最下位ビットの退化した速度タイが原因であり、RNG を ChaCha8 に統一して
根治した。今後は速度学習マイルストーン (実速度差) にもこの公平な速度タイ基盤が使える。
