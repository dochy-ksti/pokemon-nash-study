# 強 checkpoint 同士の評価一致度 (重み取り替えテスト)

## 目的

「強い checkpoint は既にナッシュ均衡に達しているのでは」という仮説の検証。
均衡に収束しているなら、独立に学習した強モデル同士は同一局面で重みを取り替えても
ほぼ同じ評価 (policy / value) を返すはず。これを直接測る。

## 方法

新ツール `eval-ckpt-agreement` を実装 (`poke_ai3_train/eval_ckpt_agreement.py`):

1. 先頭 checkpoint で自己対戦を回し局面集合を収集 (学習なし)。
2. 同一局面すべてを各 checkpoint の重みで推論し直す。
3. checkpoint ペアごとに policy 総変動距離 TV (合法手で再正規化した分布の 0.5*L1) と
   value (盤面勝率予測) の絶対差 MAE を集計。局面ごとの value 標準偏差も出す。

混合戦略均衡では best response が複数あり policy は一意とは限らないため、より確実な
「同じ評価」の指標は value の一致と位置づける。

## コマンド

```bash
cd poke-ai3-python
make eval-ckpt-agreement ARGS="\
  --checkpoint ../data/poke-ai3/weak/weak_s1_ep260.pt \
  --checkpoint ../data/poke-ai3/seed2/weak_s2_ep200.pt \
  --checkpoint ../data/poke-ai3/seed2/normal_s2_ep200.pt \
  --checkpoint ../data/poke-ai3/weak/weak_s1_ep20.pt \
  --num-games 8 --sim-concurrency 4 --sims 32 --num-batches 4 \
  --search-turn-min 4 --search-turn-max 8 --no-random --no-crit --stage 3b \
  --max-batch-size 32"
```

比較対象: 強 = weak_s1_ep260 (seed1/weak), weak_s2_ep200 (seed2/weak),
normal_s2_ep200 (seed2/strict)。対照 (弱) = weak_s1_ep20。
局面収集は先頭 weak_s1_ep260 で実施 (n_item=876)。

## 結果

```
ペア                                  policy_TV   value_MAE
weak_s1_ep260 vs weak_s2_ep200          0.0570     0.0432
weak_s1_ep260 vs normal_s2_ep200        0.0838     0.1206
weak_s1_ep260 vs weak_s1_ep20           0.2125     0.1297
weak_s2_ep200 vs normal_s2_ep200        0.0768     0.1407
weak_s2_ep200 vs weak_s1_ep20           0.2113     0.1328
normal_s2_ep200 vs weak_s1_ep20         0.2609     0.1737

局面平均 value 標準偏差 = 0.0898
```

## 解釈

- **policy は強同士でよく一致**: 強 vs 強の policy_TV は 0.057〜0.084 と小さく、
  強 vs 弱 (ep20) の 0.21〜0.26 に対し明確に小さい。着手分布 (best response) は
  独立学習でも共通の領域に収束しており、仮説を支持する。
- **value は完全には一致していない**: 同手法 (weak) 同士の weak_s1_ep260 vs
  weak_s2_ep200 は value_MAE=0.043 と非常に近いが、normal_s2 (strict) が絡むと
  value_MAE が 0.12〜0.14 に増え、弱モデル ep20 との差 (0.13) と同程度。
  盤面勝率予測 (value head のキャリブレーション) は手法 (weak/strict) によって
  なお差が残る。
- 局面平均 value 標準偏差 0.090 は「ほぼ同評価」と言い切るには大きめ。
  policy はほぼ収束、value はまだ均衡的に一致しきっていない、というのが現状。

## 結論

着手分布レベルでは強 checkpoint 同士は共通の応答に収束 (均衡近傍を支持)。
ただし盤面評価 (value) は手法差が残り、特に strict 系は weak 系と評価がずれる。
「完全に同一均衡」と言うには value の不一致が無視できない。

## 備考 / 次の一手候補

- 同手法・別 seed (weak_s1 vs weak_s2) に絞ると value も近い (0.043) → 残差の主因は
  weak/strict の手法差の可能性。weak のみ複数 seed で揃えて再測すると切り分けられる。
- 局面を normal_s2 で収集して再測すると、収集側バイアス (今回 weak_s1_ep260 が
  訪れる局面分布) の影響を確認できる。
