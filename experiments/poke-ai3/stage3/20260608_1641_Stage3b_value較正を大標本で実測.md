# Stage3b value較正を大標本で実測 (inspect-matchup)

## 目的

ハンドオフ(`docs/poke-ai3/20260608_1633_...`)の「次にやること 1」。
前回の quick run(~32試合相当)では各HPセルの realized がノイジーだったため、
標本を増やして対面×HP別の `model_value − realized`(過大/過小評価マップ)を
きれいに見る。

## コマンド

```bash
cd poke-ai3-python
uv run inspect-matchup --checkpoint-path ../data/poke-ai3/stage3b_long.pt \
  --stage 3b --num-games 16 --num-batches 16 --sims 64 --sim-concurrency 16 --no-random --no-crit
```

チェックポイント `stage3b_long.pt`(training_step=300)。学習せず(非上書き)、
自己対戦 trajectory を 6454 item 収集して対面×HP別に集計。

## 結果

### 集計 value(model / target / realized = 実勝率)

| 対面 | model | target | realized | model−realized |
|---|---|---|---|---|
| SW_v_Cl (SE有利) | 0.604 | 0.608 | 0.521 | +0.083 |
| SW_v_Go (半減不利) | 0.595 | 0.615 | 0.508 | +0.087 |
| BD_v_Cl (等倍) | 0.628 | 0.621 | 0.515 | +0.113 |
| BD_v_Go (SE有利) | 0.578 | 0.607 | 0.523 | +0.055 |

- **model ≈ target が全対面で成立** → 学習ギャップ無し(前回結論を大標本で再確認)。
- 過大評価は対面によらず +0.06〜+0.11 でほぼ一定。原因は target 自体
  (= 手別勝率の最大値 = 最善手前提)の楽観バイアス。

### HP別の過大評価マップ(新規知見)

過大評価は「自分中HP × 相手満タン(my_mid/opp_hi)」セルに集中:

- BD_v_Cl my_mid/opp_hi: model 0.65 vs realized 0.38 (+0.27, n=365)
- BD_v_Go my_mid/opp_hi: 0.44 vs 0.20 (+0.24, n=56)
- SW_v_Go my_mid/opp_hi: 0.59 vs 0.41 (+0.18, n=64)
- SW_v_Cl my_mid/opp_hi: 0.62 vs 0.53 (+0.09, n=237)

逆に勝勢セル(opp_lo)はよく較正され、満タン互角(my_hi/opp_hi)も誤差小。
=「自分が削られ・相手が満タン=不利進行」の局面で、最善手前提の target が
回復を楽観しすぎている。劣勢ほどバイアスが膨らむ。

### 交代率(参考、policy)

前回と一致。SE対面は攻撃(交代~0.22–0.24)、非SE対面は交代~0.69–0.77。
model/target/selection/chosen が全て一致し、policy 学習ギャップも無し。

## 結論

1. value の学習ギャップは無い(model≈target)。バイアスは target の構造的なもの
   (最善手前提・最大値採用)で、劣勢進行セルで最大 +0.27 に達する。
2. 較正したいなら value head の学習対象ではなく target 生成(lookahead の
   value 集約方法)側を見るべき。例: 最大値ではなく policy 加重平均にする等。
   ただし強さに直結する根拠は未確認(過大評価は方策選択には影響しない可能性)。
