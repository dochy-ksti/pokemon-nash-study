# モデルサイズ A/B (ep100 事前スクリーニング)

## 目的

hidden_size を 128→256→512→1024 と増やし、実行速度と強さの変化を見る。
まず全サイズ ep100 で速度と傾向を把握し、有望なものを ep200 へ延長する方針。

## 前提実装

`train-loop` にモデルサイズ系 CLI を追加 (このセッション):
`--hidden-size` / `--intermediate-size` / `--num-layers` / `--num-attention-heads`。
新規学習時のみ有効 (既存 checkpoint 再開時は保存設定を優先し警告)。設定は checkpoint に
保存され、eval 系ツールは保存設定で自動ロードする。

スケール方針: heads のみ比例 (head_dim=32 固定: 256→8, 512→16, 1024→32)、layers=2 固定。
他のハイパラは全サイズ共通。

| サイズ | hidden | intermediate | heads | layers |
|---|---|---|---|---|
| h128 (baseline) | 128 | 256 | 4 | 2 |
| h256 | 256 | 512 | 8 | 2 |
| h512 | 512 | 1024 | 16 | 2 |
| h1024 | 1024 | 2048 | 32 | 2 |

## コマンド

学習 (サイズ部分以外は全共通, battle-seed 固定で公平化):

```bash
cd poke-ai3-python
make train ARGS="--num-games 32 --sim-concurrency 16 --sims 64 \
  --search-turn-min 6 --search-turn-max 12 --no-random --no-crit --stage 3b \
  --max-epochs 100 --max-batch-size 341 --battle-seed 20260615 \
  --hidden-size 256 --num-attention-heads 8 \
  --checkpoint-path ../data/poke-ai3/size/h256.pt --snapshot-every 20"
```

強さ測定 (anchor_sweep, 両 side n=128/side = 合計 256):

```bash
uv run python scripts/anchor_sweep.py \
  --anchors ../data/poke-ai3/gauntlet/r2_ep105.pt ../data/poke-ai3/gauntlet/r2_ep130.pt \
  --subjects ../data/poke-ai3/size/h{128,256,512,1024}_ep100.pt --n-per-side 128
```

## 結果

### 実行速度 (学習スループット)

| サイズ | s/epoch | 対 h128 |
|---|---|---|
| h128 | 0.75 | 1.0x |
| h256 | 2.1 | 2.8x |
| h512 | 3.2 | 4.3x |
| h1024 | 7.2 | 9.6x |

h128 は ep100 を約 3 分で完走。h1024 でも約 12 分と、この小ゲームでは全サイズ実用的。

### 強さ指標 (ep100, 2 アンカー平均勝率, n=256/対戦, SE≈0.031)

| サイズ | vs r2_ep105 | vs r2_ep130 | 強さ指標 |
|---|---|---|---|
| h128 | 0.539 | 0.508 | 0.524 |
| h256 | 0.469 | 0.680 | **0.575** |
| h512 | 0.539 | 0.562 | 0.551 |
| h1024 | 0.523 | 0.461 | 0.492 |

## 解釈

- **「大きいほど強い」という単調性は ep100 時点では出ていない**。指標は
  h256 (0.575) > h512 (0.551) > h128 (0.524) > h1024 (0.492) で、最大の h1024 が最下位。
- ただしアンカー間のばらつきが大きい (h256: 0.469 vs 0.680、h1024: 0.523 vs 0.461)。
  SE≈0.031 を踏まえても 2 アンカーの不一致は非推移性 (じゃんけん) かノイズを示唆。
  ep100 の強さ指標は決定的でない。
- **h1024 は ep100 では未学習の疑いが濃い** (大型ほど収束に多エポック要する)。
  ep100 だけで「大型は弱い」と結論するのは早計。公平には ep200 まで延長が要る。

## 次の一手

- 全サイズ (少なくとも h256/h512/h1024) を ep200 まで延長し、収束後に再測。
- 最終比較は n=256/side (合計 512, SE≈0.022) に上げてノイズを下げる。
- snapshot は ep20/40/60/80/100 保存済み。延長で ep120..200 も貯まる → 強さの
  epoch 推移 (単調に上がるか) も後で描ける。
