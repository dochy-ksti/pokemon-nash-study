# Stage3b クロスチーム修正後の再学習 300ep

## 目的

`pick_config` の同チーム対戦バグ修正 (常に Team1 vs Team2 = 片側だけSE可) 後の分布で
stage3b をゼロから再学習する。汚染前学習の `stage3b_long.pt` が固定ルールに大敗
(eval-vs-rule クロスチームで 0.188 / ルール基準 0.500) した件が、汚染分布のせいか
均衡学習の本質的限界かを切り分けるため。元の `stage3b_long.pt` は保全し新パスへ。

## コマンド

```bash
cd poke-ai3-python
uv run train-loop --num-games 32 --sim-concurrency 16 --sims 64 \
  --search-turn-min 6 --search-turn-max 12 --no-random --no-crit --stage 3b \
  --max-epochs 300 --chunk-threshold 512 \
  --checkpoint-path ../data/poke-ai3/stage3b_long_xteam.pt
```

元の stage3b_long (200+100ep) と同条件。違いは pick_config 修正 (クロスチーム限定) と
新チェックポイント `stage3b_long_xteam.pt` へ 300ep 一括。

## 結果

学習完了 (training_step=300, `stage3b_long_xteam.pt`)。eval-vs-rule (n=512):

```bash
uv run eval-vs-rule --checkpoint-path ../data/poke-ai3/stage3b_long_xteam.pt \
  --stage 3b --num-games 16 --num-eval-games 512 --sims 64 --sim-concurrency 16 --no-random --no-crit
```

| モデル | 全体 | 有利(SE可) | 不利 |
|---|---|---|---|
| 汚染学習 stage3b_long       | 0.188 | 0.215 | 0.160 |
| クリーン再学習 stage3b_long_xteam | **0.564** | **0.576** | **0.553** |
| 参考: rule vs rule 基準      | 0.500 | 0.502 | 0.498 |

### 結論
1. **0.188 の大敗はほぼ全て pick_config の同チーム汚染が原因**。クリーンなクロスチーム分布で
   再学習しただけで 0.564 に回復。
2. **固定ルール基準 (0.50) を上回る (0.564)**。まともに学習すれば、学習 AI は決定論的な相手を
   単純な best-move より上手く搾取できる (均衡学習は固定相手にもプラスに働く)。
3. **有利対面で勝ち越し (0.576)、不利でも勝ち越し (0.553)**。「先発有利のほうが有利」という
   直感が汚染除去後はきちんと成立。

### 一連の決着
疑わしい評価結果 (0.47, 有利で負け越し) → 学習分布のバグ (同チーム対戦) を発見 →
pick_config 修正・再学習 → 直感どおりの健全な結果 (0.564, 有利で勝ち越し)。
ユーザーの「信じがたい」直感がバグ発見につながった。

### 補足 (初手着手別)
SW_v_Cl(有利): 初手攻撃0.531 / 初手交代0.702。BD_v_Go(有利): 初手攻撃0.577。
有利でも初手交代が有効なケースがあるのはピボット力学を反映 (どちらも勝ち越し圏)。
