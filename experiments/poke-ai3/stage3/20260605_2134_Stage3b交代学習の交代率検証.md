# Stage3b 交代学習の交代率検証

JST: 2026-06-05 21:34

## 目的

行動空間を `ACTION_DIM = NUM_MOVES + (MAX_PARTY-1)` へ拡張した縦切り(コミット済み)を
実機学習で検証する。非対称2チームの Stage3b で、**不利対面での適正交代率 (`switch_rate`)
が >0.90** に達するかを定量判定する。到達後 `battle-review` で定性確認する。

不利対面の定義(`diagnostics.stage3b_switch_diagnostics`):
自分の active 技が相手 active に SE を通せず、生存控えの技は SE を通せる状況。
そのとき適正手は控えへの交代 (argmax >= NUM_MOVES)。

## コマンド

```bash
cd poke-ai3-python
uv run train-loop --num-games 32 --sim-concurrency 16 --sims 64 \
  --search-turn-min 6 --search-turn-max 12 --no-random --no-crit --stage 3b \
  --max-epochs 20 --chunk-threshold 512 \
  --checkpoint-path ../data/poke-ai3/stage3b.pt
```

引数の意図:
- `--stage 3b`: 交代学習(非対称2チーム、各個体1技)。
- `--no-random --no-crit`: 決定論フェーズ。
- `--search-turn-min 6 --search-turn-max 12`: 交代で対局が長引くため 1v1 より深め。
- `--num-games 32 / --sim-concurrency 16 / --sims 64`: 標準スループット。
- `--max-epochs 20`: 学習更新20回で switch_rate の立ち上がりを見る。
- `--chunk-threshold 512`: 1更新あたりのサンプル蓄積閾値。

判定指標: ログの `switch_rate=...(n=...)`。> 0.90 を合格とする。

## 結果

- 全 20 エポックで **`switch_rate=0.0000`(n=86〜204)**。不利対面のサンプルは十分あるが、
  学習後モデルの greedy 着手は一度も交代を選ばなかった。
- `win_rate` は ~0.5 で安定(対称自己対戦として期待どおり)。マッチアップ別勝率も ~0.5 周辺。
- workspace は緑、学習自体は exit 0 で完走。チェックポイント: `data/poke-ai3/stage3b.pt`。

### 解釈(2仮説)

1. **均衡効果(grill-me 質問4 の懸念の実証)**: 両陣営対称で交代が先に解決されるため、
   不利側が交代しても相手も最適応答すると優位が相殺され(switch ループ)、自己対戦
   均衡では交代の優位が限界的。結果、全状態の多数を占める「攻撃が正解の局面」が
   攻撃プライアを支配し、希少な交代信号が埋もれて switch_rate=0 になりうる。
2. **縦切りのバグ**: lookahead 教師は交代を favor している(単体テスト
   `stage3b_unfavorable_matchup_favors_switch` は緑)のに、自己対戦データ生成や
   Python 学習で交代信号が伝わっていない可能性。

解析上は「不利なら交代」が各局面で有利に見える(SE 被弾回避 + 次ターン SE 設定)ため、
ちょうど 0 は仮説2(バグ)も否定できない。次は trajectory dump / battle-review で
**自己対戦の不利対面における教師 target_pi が実際に交代へ寄っているか**を確認する。

### 追加計測 — 教師選好とモデル交代確率 (決着)

一時計装で、不利対面における教師 target_pi の交代質量とモデル softmax 交代確率を測定:

| epoch | n | greedy_switch | model_switch_prob | teacher_switch_prob |
|---|---|---|---|---|
| 1 | 85 | 0.000 | 0.267 | 0.274 |
| 2 | 127 | 0.000 | 0.264 | 0.304 |
| 3 | 178 | 0.000 | 0.253 | 0.229 |
| 4 | 110 | 0.000 | 0.232 | 0.265 |
| 5 | 179 | 0.000 | 0.240 | 0.245 |

**結論: 縦切りは完全に機能している。**
- モデルの交代確率 (~0.24–0.27) が教師 (~0.23–0.30) をほぼ完全に追従。env→lookahead→
  学習→モデルの配線は端から端まで正常で、行動空間拡張は交代を正しく学習可能にしている。
- `greedy_switch_rate=0` は、~25/75 の混合教師に argmax をかけた**指標のアーティファクト**
  (交代0.25 < 攻撃0.75 なので greedy は常に攻撃)。
- 「>90% greedy 交代」は不適切な物差し。Stage3b の自己対戦均衡は混合戦略 (交代 ~25%) で、
  これは grill-me 質問4 の switch ループ懸念の実証。**正しい判定 = model_switch_prob ≈
  teacher_switch_prob (混合戦略の忠実な再現)** で、本実験はこれを満たす。

### 反映

- 診断 `stage3b_switch_diagnostics` を **交代確率ベース** (model/teacher) に作り直し、
  ログ出力も `switch_prob[model=.. teacher=..]` に変更 (コミット)。

### 残課題 (任意)

- ~25% が真の混合均衡 (1a) か、lookahead が浅く交代を過小評価 (1b) かの切り分けは未確定。
  必要なら sims↑/探索深↑で teacher_switch_prob が上がるか確認する。slice の正しさには影響しない。
