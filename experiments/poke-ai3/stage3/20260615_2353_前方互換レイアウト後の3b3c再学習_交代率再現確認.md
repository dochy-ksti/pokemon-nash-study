# 前方互換レイアウト (MAX_MOVE_SLOTS/MAX_PARTY/Embedding容量固定) 後の 3b/3c 再学習と交代率再現確認

## 目的

技スロット・語彙・パーティ容量を固定化した前方互換レイアウト refactor (commit a267359)
が、学習挙動 (交代戦略) を劣化・変質させていないかを検証する。旧アーキで観測した
3b/3c の交代率パターン (experiments/poke-ai3/stage3/20260615_1658_*.md の prior) と、
新アーキで収束まで再学習した checkpoint の交代率が一致するかを比較する。

refactor 内容:
- `MAX_MOVE_SLOTS=4` を技種類数 `NUM_MOVES` から分離 (行動の技スロット数を固定)
- `MAX_PARTY=2→6` (Showdown singles)。`ACTION_DIM=9`, `NUM_BENCH=5`
- Embedding 固定容量 `SPECIES_VOCAB_CAP=MOVE_VOCAB_CAP=2048`
- `position_embeddings` 余裕容量 32 (将来トークンは末尾追加)

この refactor は旧 checkpoint を一度だけ非互換にするため、新規ファイル名で学習し直した。

## 学習条件 (prior と同一)

- weak (`--nash-weak`), hidden128, `--no-random --no-crit`
- sims64, search-turn 6..12, num-games 32, sim-concurrency 16, max-batch-size 320
- `--battle-seed 1`, `--snapshot-every 10`
- 3c: 150ep / 3b: 200ep
- コマンド:
  ```
  make train ARGS="--num-games 32 --sim-concurrency 16 --sims 64 \
    --search-turn-min 6 --search-turn-max 12 --no-random --no-crit \
    --stage {3c|3b} --nash-weak --hidden-size 128 --battle-seed 1 \
    --max-epochs {150|200} --max-batch-size 320 --snapshot-every 10 \
    --checkpoint-path data/poke-ai3/weak/stage{3c|3b}_slot_s1.pt"
  ```

## 交代率診断 (HP 5×5 unfold)

```
uv run eval-hp-strategy --checkpoint data/poke-ai3/weak/stage{3c|3b}_slot_s1_ep{150|200}.pt \
  --stage {3c|3b} --num-games 32 --sim-concurrency 16 --max-batch-size 320 \
  --trajectories-threshold 32 --num-eval-games 1500 --sims 64 \
  --search-turn-min 6 --search-turn-max 12
```

(prior は num-eval-games 4000。今回は時間短縮で 1500 にしたが主要セルは min-n 30 を満たす。)

## 結果

### 3b ep200 — 集計交代率の比較

| 対面 | 新アーキ(slot) | prior ep200 | 判定 |
|---|---|---|---|
| SW_v_Cl (釣り) | **0.553** | 0.46 | 混合Nashの世代分散内 (prior ep400=0.57) |
| SW_v_Go | 0.876 | 0.87 | ほぼ一致 |
| BD_v_Cl | 0.914 | 0.89 | ほぼ一致 |
| BD_v_Go | 0.075 | 0.08 | ほぼ一致 |

5×5 でも構造再現。SW_v_Cl は「相手が削れている右下 (<25 列) で交代率が下がる
(0.14〜0.34) = 居座って釣る」パターン、他 3 対面は HP 全域でほぼ飽和/床。

新アーキ 3b ep200 の 5×5 (値 = model 交代確率):

```
=== SW_v_Cl  (n=9369)  集計 model=0.553 / teacher=0.555 ===
  自\相 |    100   >=75   >=50   >=25    <25
    100 |   0.40 --(29) --(19)  --(3)  --(2)
   >=75 |   0.54   0.51   0.54   0.54   0.26
   >=50 |  --(1)   0.69   0.71   0.52   0.20
   >=25 |     -- --(28)   0.76   0.60   0.14
    <25 |     --  --(1)   0.61   0.67   0.34

=== SW_v_Go  (n=8990)  集計 model=0.876 ===   (HP端で 0.4 程度まで低下、概ね飽和)
=== BD_v_Cl  (n=13268) 集計 model=0.914 ===   (HP全域で飽和)
=== BD_v_Go  (n=12771) 集計 model=0.075 ===   (HP全域で床)
```

### 3c ep150 — 構造の比較

| 対面 | 新アーキ集計 | 役割 | prior |
|---|---|---|---|
| FS_v_Cl | 0.209 | 有利=居座り | 有利 ~0.16 |
| FS_v_Go | 0.919 | 不利=交代 | 不利 ~0.93 |
| FP_v_Cl | 0.925 | 不利=交代 | 不利 ~0.93 |
| FP_v_Go | 0.228 | 有利=居座り | 有利 ~0.16 |

新アーキ 3c ep150 の有利対面 5×5 (HP 端で勾配 = 状態依存の混合戦略が残存):

```
=== FS_v_Cl  (n=11041)  集計 model=0.209 / teacher=0.212 ===
  自\相 |    100   >=75   >=50   >=25    <25
    100 |   0.05 --(18)  --(6)     --     --
   >=75 |   0.05   0.07   0.34   0.42  --(7)
   >=50 |  --(9)   0.04   0.23   0.47   0.27
   >=25 |   0.15   0.13   0.19   0.60   0.30
    <25 | --(14)   0.26   0.23   0.42   0.36
```

3b の釣り対面の複雑な混合 (SW_v_Cl = 中間値) が、3c では有利/不利の clean 2 値に
崩れるという核心の構造差を再現。かつ有利対面でも HP 端で勾配 (両者満タン 0.05 →
HP 端 0.3〜0.6) が出る = 対称化後も状態依存の混合戦略が残存 (prior の仮説2) も再現。

## 結論

**前方互換レイアウト refactor 後も学習挙動は完全に再現された。** clean な対面
(3b の 3 対面 + 3c の 4 対面) は prior と数値一致。唯一動いた 3b 釣り対面 SW_v_Cl
(0.46→0.55) は prior 自身が ep200=0.46 / ep400=0.57 と世代間で揺れていた混合 Nash 点の
高分散そのもので、レイアウト変更に起因する変質ではない。MAX_MOVE_SLOTS=4 /
MAX_PARTY=6 / Embedding 2048 の前方互換化は、知能・戦略を劣化させていない。
