# Rust エンコード経路での stage 3a 30 epoch 学習

実施: 2026-06-12 12:06 JST

## 目的

観測転送の Rust エンコード化 (bbe5ec2、ベンチ 20260612_0022) 後の新経路で、
正解ありの stage 3a が従来どおり収束することを確認しつつ、深い設定
(sims 512 / W 256 / batch 8192) での学習を実時間で確かめる。

## コマンド

```bash
cd poke-ai3-python
uv run train-loop --num-games 32 --sim-concurrency 256 --sims 512 \
  --search-turn-min 4 --search-turn-max 8 --no-random --no-crit --stage 3a \
  --max-epochs 30 --max-batch-size 8192 \
  --checkpoint-path ../data/poke-ai3/stage3a_fast30.pt
```

ログ: `logs_stage3a_fast30.log` / チェックポイント: `data/poke-ai3/stage3a_fast30.pt`

## 結果

- 所要時間: **32.3 秒 (30 epoch)** ≒ 1.08 s/epoch。
  旧 JSON 経路の計測 B (同設定で 80.7 s/epoch) では 40 分相当だった規模。
- correct_action_rate: epoch1 0.429 → **epoch4 以降 1.0000 で安定** (5〜30 すべて 1.0)。
- stage 3a (正解あり) は新経路でも従来同様に即収束。学習挙動の退行なし。

## 結論

Rust エンコード経路で学習が正常に回り、深い設定でも 30 epoch が約 30 秒。
次は stage 3b の 30 epoch (stage3b_fast30.pt) と eval-vs-rule での
現行 stage3b_long_xteam.pt (0.564) 比較。
