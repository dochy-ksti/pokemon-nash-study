# minibatch 256 vs 512 強さ A/B (step3c)

実施: 2026-06-26 12:58 JST (run 03:26–03:57 頃、funnel A→B→rate)

## 目的

[step3](20260625_2321_minibatch_スループットと強さ非劣化AB_step3.md) の留保だった
「minibatch 512 (examples/s 1.99x) が採用済み 256 に対して強さ非劣化か」を直接 A/B で検証する。
step3b は 64 vs 256 で 256 が僅差 (ノイズ圏内) だったため、さらに勾配 step が半減する 512 が
見合うかを決める。

本 run は funnel ブロック学習化 (GPU 0% 谷の改修, train-block-epochs) 後の初の本番 A/B。

## 手法

driver: `scripts/run_ab_minibatch.sh` を `MBS_A=256 MBS_B=512` で実行。
両手法とも採用設定 g64/b512/th128 + search A (depth-skew 2.0 / search 4-8 / sims 32 /
stage 3b) で共通、差分は `--train-minibatch-size` のみ (A=256 / B=512)。
funnel: epochs-per-step 5 / train-block-epochs 50 / warmup-steps 10 / finalists 3。
head-to-head は in-process 化済み (commit 65f5dff)、eval num-games 256 / 先後 512+512=1024。

## 結果

最終生存: A = {A_ep75, A_ep150, A_ep250}、B = {B_ep140, B_ep195, B_ep270}。

| ckpt | 手法 | レート |
|---|---|---:|
| A_ep250 | A (mbs256) | +12.8 |
| A_ep75 | A (mbs256) | +12.7 |
| B_ep270 | B (mbs512) | +4.6 |
| A_ep150 | A (mbs256) | -8.9 |
| B_ep195 | B (mbs512) | -9.3 |
| B_ep140 | B (mbs512) | -11.9 |

手法平均: **A (mbs256) = +5.5 / B (mbs512) = -5.5 → 勝者 A**

## 結論

- **mbs512 は非劣化ゲートを満たさない。mbs256 より明確に弱い**。
  - 差 ±5.5 Elo は、これまでの A/B (step2 ±1.5 / step3b ±1.3, いずれもノイズ圏内) の 3-4 倍。
  - 上位 2 個が両方 mbs256。mbs512 の最良 (B_ep270) でも +4.6 止まりで、残り 2 個は最下位群。
    過去 A/B は極値が手法をまたいで混在 (手法間差 < ckpt 間ばらつき) だったが、今回は
    mbs512 が一貫して下位 = コヒーレントな劣化シグナル。
  - クロス対戦も A 優勢が多数 (A#2 vs B#0 558/466、A#0 vs B#1 565/459 等)。
- **mbs512 は不採用。mbs256 を維持**。examples/s +18% (256→512) は勾配 step 半減
  (1更新 mbs256≈24 step → mbs512≈12 step) による強さ低下に見合わない。

## minibatch スイープの最終結論 (step3 系まとめ)

| minibatch | examples/s (vs 64) | 強さ判定 | 採用 |
|---|---:|---|---|
| 64 | 1.00x | 基準 | - |
| 256 | 1.69x | vs64 非劣化 (step3b, ±1.3) | **採用** |
| 512 | 1.99x | vs256 劣化 (step3c, A+5.5/B-5.5) | 不採用 |

最終採用既定 (train-loop): `--minibatch-size 256` (commit 6c93d41 で反映済み, 据え置き)。

## 副次知見: ブロック学習化の効果

- funnel が train-block-epochs 50 / snapshot-every 5 で「50ep を 1 プロセス連続学習し 5ep 毎に
  snapshot」する方式に改修済み。subprocess 再起動が 1/10 に減り、GPU 0% 谷を大幅削減。
- 本 run はその初の本番適用。batch_stats は avg_real ~457-465/512、empty_ratio ~0.09-0.11 で
  b512 が良く充填され、学習中の GPU 供給も安定。A/B 結果の妥当性 (mbs256/512 とも正常学習) を確認。

## 再現ラン (v2, 2026-06-26 04:45 JST)

別シード (battle_seed は未指定時ラン毎ランダム) で同条件 (g64/b512/th128 + search A,
epochs-per-step 5 / train-block-epochs 50 / warmup 10 / finalists 3, 差分は minibatch のみ)
の独立ランをもう一度実行し、差が再現するか確認した。

| | v1 (初回) | v2 (再現) |
|---|---:|---:|
| A (mbs256) 手法平均 | +5.5 | **+4.8** |
| B (mbs512) 手法平均 | -5.5 | **-4.8** |
| 手法間差 (Elo) | 11.0 | 9.6 |

- v2 レート: A#1 +11.9 / B#2 +4.4 / A#0 +2.7 / A#2 -0.3 / B#1 -4.6 / B#0 -14.0。
- 順位構造も再現: **最上位は A(mbs256) の A#1、最下位は B(mbs512) の B#0**。
  トップ A#1 は B 全3個に勝ち越し (vs B#1 583/441=56.9%, vs B#0 517/507, vs B#2 516/508)。
  mbs512 の最良 (B#2 +4.4) でも A 中位に届かず。
- クロス生勝率も A 優勢が再現 (A#0 vs B#0 574/450=56.1%, A#1 vs B#1 583/441=56.9%)。
- **2ラン平均で手法差 ±5.1 Elo**。過去のノイズ A/B (±1.3〜1.5) の 3 倍以上で安定。
  「mbs512 は mbs256 より一貫して弱い」は**再現性あり**と確定。結論 (mbs256 維持) は不変。
