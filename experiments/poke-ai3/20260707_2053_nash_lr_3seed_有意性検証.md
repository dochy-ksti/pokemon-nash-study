# nash_learning_rate 3seed 複製 — 1.25/1.5/2.0 の有意差検証

策定/実行: 2026-07-07 20:53 JST 開始 (背景実行)

## 目的・仮説

先行 A/B (`20260707_1816_nash_lr_2.0_vs_1.5_A_B_K1r0.5.md`) は各アーム1 funnel run のみで、
1 run=同系譜3 finalist が相関するため run 単位ノイズが手法平均を支配し、順位 1.5>1.25>2.0 は
**有意判定不能**だった。本実験は各 nash_lr を **3 run ずつ (計9 funnel)** 複製し、run-avg を
独立標本 (アームあたり n=3) として method 間に本当に差があるかを見る。

- battle_seed は funnel が None を渡し train_loop 側で `secrets.randbits(64)` により毎回
  ランダム化される。よって同一コマンドの反復で独立 seed が得られる。
- 全 run は `shared_init.pt` を共有し `--nash-learning-rate` だけを変える。他フラグは
  先行 A/B と完全一致 (K1 r0.5 現行既定構成)。

## 手法 (計9 funnel run→各3 finalist)

| tag | nash_lr | seed |
|---|---:|---|
| NLR12_s1/s2/s3 | 1.25 | 各ランダム |
| NLR15_s1/s2/s3 | 1.5  | 各ランダム |
| NLR20_s1/s2/s3 | 2.0  | 各ランダム |

## 実行方針

切断分離ドライバ [run_nash_lr_3seed.sh](../../poke-ai3-python/scripts/run_nash_lr_3seed.sh)
(setsid+nohup, 冪等: 既存 `*_finalists.json` はスキップ) で 9 funnel を1本ずつ順次実行し、
最後に9 finalists.json (27 finalist) を1プール総当たり rate。成果物は public TDIR。

### 起動コマンド

```bash
cd poke-ai3-python
setsid nohup bash scripts/run_nash_lr_3seed.sh > /tmp/nash3/driver.log 2>&1 &
```

### 各 funnel (tag と nash_lr 以外は共通。以下は NLR20_s1 の例)

```bash
uv run python scripts/ckpt_tournament.py funnel --tag NLR20_s1 \
  --start "$PUB/shared_init.pt" --enemy-window 1 --self-play-ratio 0.5 \
  --nash-learning-rate 2.0 \
  --depth-skew 2.0 --search-turn-min 4 --search-turn-max 8 \
  --sims 64 --sim-concurrency 16 --train-num-games 64 --train-max-batch-size 512 \
  --train-trajectories-threshold 128 --train-minibatch-size 256 \
  --epochs-per-step 5 --train-block-epochs 50 --warmup-steps 10 \
  --peaks-per-rr 3 --finalists-target 3 --max-added-epochs 1000 \
  --n-per-side 512 --num-games 256 --stage 3b --random --crit
```

### 評価 (9 run 完了後 rate 1回)

```bash
uv run python scripts/ckpt_tournament.py rate \
  --funnel-json "$PUB"/NLR12_s{1,2,3}_finalists.json \
                "$PUB"/NLR15_s{1,2,3}_finalists.json \
                "$PUB"/NLR20_s{1,2,3}_finalists.json \
  --n-per-side 512 --num-games 256 --stage 3b --random --crit
```

## 所要 (見積り)

先行実測で funnel 1本 ~56–70分。9本で **約9–11時間** + rate 数十分 (27 finalist 総当たり)。
RTX 5090 単機・背景実行。

## 結果 (2026-07-07 完了)

9 funnel 完走 (各 ~55–90分, 12:00→21:29 UTC 通し) → 27 finalist 1プール総当たり (n-per-side 512)。

### nash_lr アーム別 (各9 finalist, Bradley-Terry 平均0 Elo)

| nash_lr | アーム平均 | run平均 (s1/s2/s3) | 個体レンジ |
|---|---:|---|---|
| 1.5  | **+0.83** | +1.8 / −4.8 / +5.6 | −9.2 〜 +10.1 |
| 1.25 | +0.14 | −1.1 / −0.1 / +1.6 | −5.4 〜 +3.8 |
| 2.0  | −1.02 | +1.9 / +3.8 / −8.7 | −9.7 〜 +12.6 |

### 有意性 (run平均を独立標本 n=3 とした一元配置ANOVA)

- アーム平均の広がりは **1.85 Elo** (+0.83〜−1.02) に過ぎない。
- 一方 run 間 SD は 1.4 (1.25) / 5.3 (1.5) / 6.7 (2.0) Elo と桁違いに大きい。
- **F(2,6) ≈ 0.11, p ≈ 0.90** → 手法差は完全にノイズ。有意差なし。

### 結論

- 先行 A/B (各1本) の「2.0 が3 finalist 全滅で最弱」は**再現しない**。運の悪い1 run を
  引いていただけ。今回 2.0 は全体最高個体 NLR20_s2#1 (+12.6) すら産出しており、s3 の
  破滅 run (−8.7) が平均を落としているだけ。
- どのアームも「1本良く1本悪い」で run 単位ノイズ (battle seed + 学習確率性) が手法差を
  完全に飲み込む。先行実験で警告した「1 run=同系譜3 finalist は相関」の罠が、複製によって
  **F≈0.11 と定量的に裏付けられた**。
- **nash_lr 1.25/1.5/2.0 の優劣は統計的に区別不能。既定 nash_learning_rate=2.0 を変える
  根拠はなく維持。**「1.5 が良さそう」の探索的示唆も棄却。
- 副次的教訓: この funnel 構成では 1 funnel run の run-avg SD が最大 ~6.7 Elo あり、
  数 Elo 規模のハイパラ差を検出するには各アーム 3 本でも足りない (信頼区間が広すぎる)。
  今後この系での小差比較は費用対効果が低い。

## 追加分析: フロア (最悪ケース勝率) によるナッシュ近接度

「対称ゼロサム評価では、ナッシュ均衡戦略は誰を相手にしても勝率が価値 (=50%) を下回らない
(unexploitable)」という性質に着目し、平均レート (BT) ではなく各個体の **フロア = 全26相手に
対する最低勝率** を評価軸にした。フロアが50%に近いほど搾取されにくく均衡に近い。

同じ rate.log の全351ペア (side入替込み各1024戦, 引分0.5) から27×27勝率行列を復元して算出。

### フロア順 上位/下位 (floor=最低勝率, mean=平均, ceil=最高)

| 順 | checkpoint | nash_lr | floor% | mean% | ceil% | 最悪の相手 |
|---:|---|---:|---:|---:|---:|---|
| 1 | NLR15_s3#1 | 1.5 | **49.0** | 51.5 | 55.3 | NLR15_s1#2 |
| 2 | NLR15_s1#2 | 1.5 | 48.1 | 51.1 | 54.2 | NLR12_s2#1 |
| 3 | NLR20_s1#2 | 2.0 | 47.7 | 50.4 | 53.9 | NLR12_s3#1 |
| 4 | NLR12_s2#2 | 1.25 | 47.0 | 50.0 | 53.6 | NLR15_s3#1 |
| 5 | NLR12_s3#0 | 1.25 | 47.0 | 50.4 | 54.3 | NLR20_s3#2 |
| … | | | | | | |
| 26 | NLR12_s3#2 | 1.25 | 39.1 | 50.3 | 58.1 | NLR20_s2#1 |
| 27 | NLR20_s3#2 | 2.0 | 37.9 | 48.6 | 56.2 | NLR12_s1#1 |

### 発見

- **平均最強 ≠ ナッシュ的**: BT最上位だった NLR20_s2#1 は mean 51.9% / ceil 60.9% と派手だが
  **floor は 43.9%** (NLR20_s1#0 に 44:56 で負ける) =「弱い相手を大きく叩くが特定相手に搾取される
  高分散型」。対して NLR12_s2#2 は mean 50.0% / floor 47.0% / ceil 53.6% と誰にもフラット=より均衡的。
  → ナッシュ近接度は BT では測れず floor で測るべき、という点がデータで裏付けられた。
- **フロアは約49%で頭打ち**: 完全に ≥50% を保証する unexploitable な個体は1つも無い
  (最良 NLR15_s3#1 でも最悪相手に 49:51)。均衡近傍ではあるが未到達。
- アーム別フロア平均は NLR15 45.8% / NLR12 44.2% / NLR20 43.5% だが、本編 ANOVA (p≈0.90) を
  踏まえるとこの差もノイズ内の可能性が高い。

### 留保

このフロアは **27個体プール内の最悪ケース**に過ぎない。27個体は同系譜で相関しており、固定個体に
対する専用の搾取AI (best response) を別途学習すれば、どの個体もこれより下 (<50%) へ叩ける可能性が
高い。真の exploitability を測るには best-response 学習が必要 (未実施)。

### 再現手順 (rate.log からの復元)

```bash
# 全351ペアの "A vs B: A勝=x B勝=y 引分=z" 行を正規表現で抽出し
# 27x27 勝率行列を構築、各行の min/mean/max を算出:
python3 - <<'PY'
import re, collections
pat=re.compile(r"^  (\S+) vs (\S+): \S+勝=(\d+) \S+勝=(\d+) 引分=(\d+)")
wr=collections.defaultdict(list)
for line in open("/tmp/nash3/rate.log"):
    m=pat.match(line)
    if not m: continue
    a,b,aw,bw,dr=m.group(1),m.group(2),int(m.group(3)),int(m.group(4)),int(m.group(5))
    tot=aw+bw+dr; ra=(aw+0.5*dr)/tot
    wr[a].append(ra); wr[b].append(1-ra)
rows=[(p,min(v),sum(v)/len(v),max(v)) for p,v in wr.items()]
for p,fl,me,ce in sorted(rows,key=lambda r:-r[1]):
    print(f"{p:14} floor={fl*100:5.1f} mean={me*100:5.1f} ceil={ce*100:5.1f}")
PY
```

可視化 (27×27ヒートマップ + ソート可能なフロア表) は Claude Artifact で作成した

## 追加検証: head-to-head の再現性 (rate を独立にもう1回)

相性関係と floor/mean/ceil が battle-seed ノイズか本物の signal かを切り分けるため、同じ9
finalists (27個体) に対し fresh random seed で `rate` をもう1回実行 (rate2.log) し、run1 と
セル単位・個体指標を突き合わせた。

### コマンド (run2)

```bash
uv run python scripts/ckpt_tournament.py rate \
  --funnel-json "$PUB"/NLR12_s{1,2,3}_finalists.json \
                "$PUB"/NLR15_s{1,2,3}_finalists.json \
                "$PUB"/NLR20_s{1,2,3}_finalists.json \
  --n-per-side 512 --num-games 256 --stage 3b --random --crit
```

### 結果: ほぼ完全再現

| 指標 | 値 |
|---|---|
| セル単位 (351ペア) 平均絶対差 | **0.11 点** |
| セル単位 RMSE | 0.14 点 |
| セル相関 (ordered 702) | **r = 0.999** |
| \|差\|>3点のペア | **0 / 351** |
| 勝敗の向き一致 | 347/351 = 98.9% |
| floor / mean / ceil の run間相関 | いずれも r = 0.999 (MAD 0.03〜0.10点) |

- 向きが食い違った4ペアはすべて 50% ちょうど付近 (49.9↔50.0 の符号反転) で実質的な相性ではない。

### 解釈: 2種類のノイズの分離 (本編 ANOVA と非矛盾)

- **評価 (固定2個体の勝率測定) はほぼ決定論的**。n-per-side 512 (=1ペア1024戦) で各セルが
  ±0.15点に収束。→ 「この個体はあの個体に 56:44 で勝つ」という相性関係・floor/mean/ceil は
  **本物の signal で再測定に揺らがない**。
- **ノイズは学習側にある**。funnel run ごとに「どんな個体が生まれるか」が run 間 SD ~6.7 Elo で
  ばらつく (本編 ANOVA p≈0.90 が飲み込まれた原因はこちら)。
- 結論: フロアランキング等 (NLR15_s3#1=49.0%で最も搾取されにくい / NLR20_s2#1=高分散型 等) は
  **この27個体に固有の確かな性質**。揺らぐのは「再学習したら別の27個体になる」部分だけ。
  → **相性・floor/mean/ceil は再現する (測定は信頼できる) が、どの nash_lr が良いかは再現しない
  (学習のばらつきに埋もれる)。**

## 後日の既定変更判断: 2.0 → 1.5 (2026-07-08)

本実験の統計的結論は「1.25/1.5/2.0 に有意差なし (F≈0.11)」であり、**性能を根拠にした変更ではない**。
その上でユーザ判断により既定 `nash_learning_rate` を **2.0 → 1.5** に変更した。理由:

- 2回の探索 (先行 A/B と本 3seed) でアーム平均・フロア平均とも 1.5 が名目上わずかに上位で、
  少なくとも 2.0 より悪いという兆候はない (ノイズ内)。
- nash_weak 穏当化版の趣旨 (Nash 追従を穏やかにして学習を安定させる) と方向が一致する、
  やや保守的な既定に倒しておく、という「いちおう」の措置。
- 有意差が無い以上どちらでも実害は無く、可逆。将来 signal が出れば再検討する。
