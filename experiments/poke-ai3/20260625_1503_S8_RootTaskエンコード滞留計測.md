# S8 RootTask エンコード滞留計測

作成: 2026-06-25 15:03 JST

## 目的

`RootTask` が `encode_batch + pack_batch` を同期実行している間に次の観測が滞留し、
エンコードを別スレッドへ移すことで S8 の実スループットが改善し得るかを判断する。

採用条件は、別スレッド版が現行版よりエンドツーエンドで 5%以上高速であること。

## 一時計測

`POKE_AI3_ROOT_TIMING=1` のときだけ、以下を集計するコードを
`poke-ai3/src/root_task.rs` に追加した。

- `encode_batch + pack_batch` の累積時間と wall 比率
- 1バッチ当たりの平均encode時間
- バッチ送出間隔
- encode完了時点の `root_receiver` 待ち行列長

通常実行では時刻計測とログ出力を行わない。

## コマンド

```bash
cd poke-ai3-python
make build
cp ../data/poke-ai3/tournament/shared_init.pt /tmp/root_timing_s8.pt

POKE_AI3_ROOT_TIMING=1 uv run train-loop \
  --num-games 32 \
  --sim-concurrency 16 \
  --sims 64 \
  --search-turn-min 4 \
  --search-turn-max 8 \
  --depth-skew 2.0 \
  --no-random \
  --no-crit \
  --stage 3b \
  --max-epochs 45 \
  --battle-seed 12345 \
  --checkpoint-path /tmp/root_timing_s8.pt
```

max batchは現行既定値の256。

## 結果

全体:

- elapsed: 62.72秒
- user: 184.11秒
- sys: 11.19秒
- 1 epoch: 1.394秒

45,000バッチ時点の安定値:

```text
root_timing wall_s=29.126 batches=45000 encode_s=7.707 encode_ratio=0.2646
avg_encode_us=171.27 avg_interval_us=647.24
avg_queued_after_encode=236.42 max_queued_after_encode=1443
```

同時点の実GPUバッチ平均は約237.9行、empty比率は約7.1%。

## 判断

- encode+packはRootTask wallの約26.5%を占める。
- encode完了時点で平均236件、ほぼ次の1バッチ分が既に待ち行列へ滞留している。
- GPU応答の配送自体はRootTaskを経由しないため、止まっているのは次バッチの形成と送出。
- 5%改善の可能性を事前に棄却できる状態ではない。別スレッド版を試作してA/Bする価値がある。

ただし26.5%全体を短縮できるわけではない。encodeを別スレッドへ移すことで追加される
チャネル通信、順序制御、CPU競合を含め、最終判断は固定seed・交互5組の中央値で行う。
