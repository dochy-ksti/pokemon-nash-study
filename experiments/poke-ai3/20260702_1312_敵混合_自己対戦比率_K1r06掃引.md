## 目的・仮説

敵混合 (`--enemy-window 1`) は K=1・自己対戦50% が現状最良 (experiments 20260701_2246, baseline 比 +9.1 Elo)。
本実験では **自己対戦比率 r を軸**に振り、`r=0.6` (自己対戦寄り) が `r=0.5` を上回るかを判定する。

仮説: 敵混合はやや薄めた方 (自己対戦60%) が、自己対戦の学習密度を保ちつつ遠い過去への頑健性も
得られてバランス良くなるかもしれない。逆に敵接触が減って効果が薄まる可能性もある。

## 実装 (本セッション)

`ckpt_tournament.py` funnel に `--self-play-ratio` フラグ追加 (type=float, default=0.5)。
`configure_enemies` を剩余バケット方式から**しきい値方式**へ変更:
- 自己対戦数 `s = round(n*r)` を先頭 gid に割当 (role 0)、残り `n-s` を敵K個へ round-robin (role 1)。
- K=1, r=0.5, n=256 → 自己対戦128/敵128 (旧 50/50 と同一 count = 後方互換)。
- K=1, r=0.6, n=256 → 自己対戦154/敵102。
- game_id は run 全体で安定スロットのため、先頭切りでも役割が run 中ブレず安定。

## 手法 (house methodology: 1手法=1 funnel run→finalist 3)

| 手法 | tag | self-play-ratio | 実行 |
|---|---|---:|---|
| K0 (純自己対戦 baseline) | RC64 | — | 流用 |
| K1 r0.5 (現行最良) | K1 | 0.5 | 流用 |
| K1 r0.6 (新規) | K1r06 | 0.6 | 新規1 run |

- 全て `--enemy-window 1` 前提。K1(0.5) は旧ハードコード 50/50 だが新 r=0.5 と割当同一 → 流用妥当。
- 新規 K1r06 は RC64/K1 と `--self-play-ratio` 以外すべて同一 (shared_init 共有)。

## パラメータ (K掃引と完全一致 + --self-play-ratio 0.6)

```bash
cd poke-ai3-python && make build && \
uv run python scripts/ckpt_tournament.py funnel --tag K1r06 \
  --start "$TDIR/shared_init.pt" \
  --enemy-window 1 --self-play-ratio 0.6 \
  --depth-skew 2.0 --search-turn-min 4 --search-turn-max 8 \
  --sims 64 --sim-concurrency 16 \
  --train-num-games 64 --train-max-batch-size 512 --train-trajectories-threshold 128 \
  --train-minibatch-size 256 \
  --epochs-per-step 5 --train-block-epochs 50 --warmup-steps 10 \
  --peaks-per-rr 3 --finalists-target 3 --max-added-epochs 1000 \
  --n-per-side 512 --num-games 256 --stage 3b --random --crit
```

## 追記: r=0.4 も測定 (2026-07-02 16:07 JST)

r=0.6 が逆効果だったため逆方向 r=0.4 (敵60%) を追試 (tag K1r04, 他同一)。
rate は RC64+K1(0.5)+K1r06(0.6)+K1r04(0.4) の全12 finalist 1プール総当たり。

| 手法 | self-play-ratio | 敵比率 | 手法平均 | 個体レート |
|---|---:|---:|---:|---|
| **K1r04** | 0.4 | 60% | **+6.7** | +7.3 / +2.3 / +10.5 |
| K1 | 0.5 | 50% | +1.2 | −5.8 / +4.4 / +5.0 |
| K1r06 | 0.6 | 40% | −3.7 | −4.4 / −3.9 / −2.8 |
| RC64 (K0) | 純自己対戦 | 0% | −4.2 | −4.7 / −2.6 / −5.3 |

**結論 (更新)**: 自己対戦比率が低い (=敵混合が多い) ほど強い、という単調傾向 (0.4 > 0.5 > 0.6)。
r=0.4 が baseline 比 +10.9 Elo で最良。r=0.6 が悪かったのは「敵を薄めた」からで、逆に
**敵を増やす方が非推移性への耐性が付く**。最適はさらに低い r の可能性 → 次は r=0.3 (敵70%) を試す。
既定 0.5 の是非は r 掃引が収束してから再判断 (現時点では 0.4 が有力)。

## 追記: r=0.3 も測定 → 最適は r=0.4 (山型) (2026-07-02 17:02 JST)

単調に見えたため r=0.3 (敵70%) を追試 (tag K1r03)。rate は RC64+K1+K1r06+K1r04+K1r03 の
全15 finalist 1プール総当たり。

| 手法 | self-play-ratio | 敵比率 | 手法平均 |
|---|---:|---:|---:|
| **K1r04** | 0.4 | 60% | **+8.4** |
| K1 | 0.5 | 50% | +0.4 |
| K1r03 | 0.3 | 70% | −1.4 |
| K1r06 | 0.6 | 40% | −2.3 |
| RC64 (K0) | 純自己対戦 | 0% | −5.1 |

**最終結論**: 自己対戦比率は **r=0.4 (自己対戦40%/敵60%) が最適 (山型)**。0.4 から 0.3 でも 0.6 でも
悪化。0.3 は敵過多で学習データ (自己対戦の P1+P2) が痩せ逆効果。r=0.4 は baseline 比 +13.5 Elo、
現行既定 0.5 比 +8.0 Elo。→ **採用済み**: `--self-play-ratio` の既定を 0.5→0.4 に変更。

## 追記: r=0.45 / 0.35 も測定 → 最適ゾーン 0.35-0.4、山頂 0.4 (2026-07-02 21:23 JST)

0.4 近傍を詰めるため r=0.45, r=0.35 を追試 (tag K1r045/K1r035)。以下は最新の全21 finalist
1プール総当たり (r=0.35 追加時)。

| 手法 | r | 手法平均 |
|---|---:|---:|

| K1r035 | 0.35 | +4.5 |
| K1 | 0.5 | +0.5 |

**結論 (確定)**: r=0.4 が3プール連続で最良 (+8.4/+8.0/+7.5)、0.35 も良好 → **最適ゾーンは 0.35-0.4、
山頂 0.4**。r=0.45 が −6.7 と最下位なのは 1ラン=3 finalist (同系譜・相関) による run 単位ノイズの
典型で、単発掃引ではこの粒度差は解像不能。**既定 0.4 は妥当、これ以上の細分化は無意味。**
厳密確定には r=0.4/0.35 の別 seed 複製で method-avg を独立標本化するのが唯一の有効な次手。

## 追記: r=0.4 再現ラン → 再現せず、比率の優劣は判定不能 (2026-07-02 22:44 JST)

r=0.4 の優位を確認するため別 battle_seed で再現ラン (tag K1r04b, shared_init 同一)。
全24 finalist 1プール総当たり:

| 手法 | r | 手法平均 |
|---|---:|---:|
| K1r04 (1回目) | 0.4 | +7.2 |
| K1r035 | 0.35 | +5.6 |
| K1r06 | 0.6 | +0.9 |
| K1 | 0.5 | +0.3 |
| K1r03 | 0.3 | −1.6 |
| **K1r04b (再現)** | **0.4** | **−3.6** |
| RC64 | 0 | −3.8 |
| K1r045 | 0.45 | −5.1 |

**最終結論 (前言撤回)**: 同一 r=0.4 の2本が **+7.2 と −3.6** に割れた (差 ~11 Elo)。r=0.4 の優位は
**再現せず、run 運 (battle seed + 学習確率性) が支配的**。K1r04b は baseline とほぼ同じ (−3.6 vs −3.8)。
**1ラン=3 finalist は同系譜で相関し独立標本にならない**ため、r 掃引の順位差は実力差ではなくノイズだった。
現データでは比率の有意差は判定不能。→ **既定を 0.4 に変えた判断を撤回し 0.5 に戻す** (素直・後方互換)。
比率を真に判定するには各 r を 3本ずつ複製し method-avg を独立標本化する必要があるが、signal の
弱さから費用対効果は低い。
