# S8 sims半分・通常・2倍 強さ比較

実施: 2026-06-25 16:43 JST

## 目的

S8 (`search-turn-min=4`, `search-turn-max=8`, `depth-skew=2.0`) の sims 数だけを変え、
学習後の強さに差が出るか比較する。

| タグ | sims | 状態 |
|---|---:|---|
| S8half | 32 | 新規funnel |
| A（通常S8） | 64 | 既存finalistsを再利用 |
| S8x2 | 128 | 新規funnel |

全構成とも共通の `shared_init.pt` を開始点とし、funnelでfinalists 3個を選抜する。

## 実行環境

- commit: `1c10ec6`
- 実験直前に `make build`
- stage: `3b`
- randomness / critical hit: 無効
- worktree上の `RootTask` 計測コードは未コミットだったが、
  `POKE_AI3_ROOT_TIMING` は設定せず実行

## funnel

新規2構成は次の共通条件で実行し、`--sims` だけを変えた。

```bash
cd poke-ai3-python
uv run python scripts/ckpt_tournament.py funnel --tag TAG \
  --start ../data/poke-ai3/tournament/shared_init.pt \
  --peaks-per-rr 3 --finalists-target 3 --warmup-steps 10 \
  --epochs-per-step 20 --max-added-epochs 4000 \
  --n-per-side 512 --num-games 64 --stage 3b \
  --depth-skew 2.0 --search-turn-min 4 --search-turn-max 8 --sims SIMS
```

### S8halfの一時ハング

S8halfの `ep740 -> ep760` 学習が、GPU使用率0%、checkpoint更新なしのまま約14分停止した。
`S8half_state.json` はep740境界まで保存済みだったためプロセスを中断し、同じ引数に
`--resume` を付けて再開した。再開後はep760を正常完了し、選抜履歴を維持したまま完走した。

## 結論

点推定の順位は **S8half > S8x2 > 通常S8** だった。ただし、
S8halfとS8x2の直接対戦は18,432試合でわずか4勝差、通常S8との差も1%未満である。
さらに各構成内のcheckpoint間ばらつきは構成平均差より大きい。

したがって今回の強さ比較からは、**simsを32・64・128のどれにしても実質的な強さは同等で、
simsを増やすほど強くなる傾向は確認できない**。少なくとも `sims=32` へ半減しても、
この条件では強さ低下は観測されなかった。

速度・計算コストを含めて採用判断するなら、強さが同等だった `sims=32` が有力候補になる。
ただし構成ごとにfinalists 3個だけなので、確証を強めるには追加seedでの再現確認が必要。
