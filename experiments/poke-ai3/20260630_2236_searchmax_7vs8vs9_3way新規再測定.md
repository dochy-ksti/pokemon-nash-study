# search-turn-max 7 vs 8 vs 9 — 3アーム新規 3way (強さと時間)

実施: 2026-06-30 11:35–13:5x JST (probe → funnel SMAX7/8/9 → rate, random+crit)

## 目的・仮説

`--search-turn-max` (rollout 最大深さ ply) を 7/8/9 で振り、強さと時間を比較する。
前回 3way ([20260630_2010](20260630_2010_searchmax8vs9_sims64vs128_強さと時間3way.md)) では
search9 が baseline(search8=RC64 流用) に対しコヒーレントに弱く見えた (手法平均 -9.2 vs +5.3、
A_s9 finalists 3 個が全体最下位)。今回は **search7 を加え、かつ 8/9 も作り直して同時期
3アーム** にし、run 間ノイズを排した上で 7/8/9 の真の差を測る。

- 3アームとも新規 funnel。差分は `--search-turn-max` のみ (min は 4 据え置き、sims 64)。
- COMMON は前回 3way と同一: shared_init / depth-skew 2.0 / search-min 4 / sims 64 /
  g64 b512 th128 / minibatch 256 / eps5 / block50 / warmup10 / finalists3 /
  max-added-epochs 1000 / stage 3b / random+crit。
- 段階的方針 (c): まず各 1 ラン。driver: `scripts/run_searchmax_3way.sh`。

## 結論

- **search-turn-max 7/8/9 に信頼できる強さ差は無い**。手法平均は -2.1〜+2.8 の幅 (3アーム
  spread 4.9 Elo) に収まり、**単発ランのノイズ帯 (±5 Elo 規模、finalists 個体差込み) と同水準**。
  順位 (9 > 7 > 8) はついたが差はノイズに埋もれる。
- **時間は深さに単調**: search7 が baseline(8) の 0.85x、search9 が 1.09x。
  rollout 1 手の伸縮で ±約 15% の生成コスト差。

### ⚠️ 前回 3way (20260630_2010) の "search9 が弱い" は否定される

- 前回は search9 が手法平均 -9.2 で baseline +5.3 比 約 -14 Elo、しかも A_s9 finalists 3 個が
  全体最下位で「コヒーレント=ノイズでない」と判定していた。
- **今回の同時期 3アーム新規ランでは search9 が逆に手法平均トップ (+2.8)、finalists は
  +1.5/+4.5/+2.3 と密にクラスタ**。一方 search8(今回新規) が -8.1/-4.0/+5.7 と大ばらつきで
  最下位個体を出した。「どのアームがコヒーレント最下位になるか」がランで入れ替わった。
- → 前回の "search9 はコヒーレントに弱い・実効の負効果" は **run 間ノイズの偶然** だった
  と訂正する。finalists 3 個が揃って下に出ても、それ自体は 1 ラン内の相関にすぎず手法の
  真の劣位を意味しない、という教訓 (epochs-per-step 実験と同型の再現失敗)。前回
  20260630_2010 で search9 の負効果に再現ランを省いた判断は誤りだった。
- baseline 維持の最終結論自体は変わらない (7/8/9 どれも強さ差なし) が、**search9 が
  害という前回の根拠は無効**。

### 留意

- 1 ラン・random+crit 単一。3アームとも今回は同時期新規なので run 間ノイズは最小化したが、
  各アーム 1 ランのため ±5 Elo の単発ノイズは残る。"差なし" 結論はその範囲で頑健。
- 強さが同等なら **search7 が最速 (baseline 比 0.85x)** なので、生成コスト削減目的でのみ
  search7 を検討する余地はある (強さは伸ばさない)。ただし前回 search9 の例があるとおり
  単発では断定しにくく、コスト削減を本採用するなら再現ラン推奨。
